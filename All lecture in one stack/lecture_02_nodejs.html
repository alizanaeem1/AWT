Node.js Kya Hai? Core Features

0 – 30 Minutes

🟢
DefinitionNode.js ek open-source, cross-platform runtime environment hai jo JavaScript ko browser ke bahar (server side) chalata hai. Google Chrome ka V8 Engine use karta hai.
⚡

Asynchronous & Non-Blocking

Node.js ek kaam karte waqt doosra kaam rok nahi deta. Multiple requests ek saath handle karta hai.

🧵

Single-Threaded Scalable

Ek hi thread hoti hai lekin event-driven model se hazaron requests manage karta hai.

🚀

Fast Performance

V8 engine JS ko seedha machine code mein convert karta hai — bahut tez!

💻

Cross-Platform

Windows, Linux, macOS — har jagah chalata hai bina modification ke.

🍽️
Real-Life Example — Restaurant Waiter Traditional (Blocking): Waiter ek order leta hai, khana aane tak rukta hai, phir agla order leta hai.
Node.js (Non-Blocking): Waiter sab ke orders leke rakhta hai, jis ka khana tayar ho use deta hai. Bahut efficient!

Node.js Modules

Reusable Code Blocks

📚
Module kya hota hai?Module ek reusable code block hai. Node.js mein har file ek module hoti hai. Jaise ek book mein chapters hote hain — har chapter ek specific topic handle karta hai!
Module Naam Type Kaam Install?
fsCoreFiles padhna/likhnaNahi – built-in
httpCoreWeb server bananaNahi – built-in

JavaScript – Module Example 
// 1. Core Module use karna
const fs = require('fs');       // File System
const http = require('http');   // HTTP Server
const path = require('path');   // File Paths

// 2. Local Module banao aur use karo
// myHelper.js mein:
module.exports = { greet: (name) => `Hello, ${name}!` };

// main.js mein:
const helper = require('./myHelper');
console.log(helper.greet('Ali')); // Hello, Ali!

// 3. Third-party module (pehle npm install express)
const express = require('express');
const app = express();

Event Loop, Callbacks & Emitters

60 – 90 Minutes

🔄

Event Loop

Node.js ka dil! Call Stack, Event Queue, aur Callback Queue continuously check karta rahta hai. Traffic police officer ki tarah!

📞

Callbacks

Ek function jo doosre function ko argument ki tarah diya jata hai. Kaam complete hone ke baad call hota hai — jaise restaurant mein buzzer!

📡

Event Emitters

Events emit (trigger) karte hain aur listen karte hain. Observer pattern par based hai. File upload, button click, network request!

JavaScript – Callback Example 
const fs = require('fs');

// Async callback — file padhna
fs.readFile('data.txt', 'utf8', (err, data) => {
  if (err) {
    console.error('Error!', err);
    return;
  }
  console.log('File mila:', data);
});

console.log('Yeh pehle chalega!'); // Non-blocking!
JavaScript – Event Emitter 
const EventEmitter = require('events');
const emitter = new EventEmitter();

// Event sunna (listen)
emitter.on('order', (item) => {
  console.log(`Order aya: ${item}`);
});

// Event trigger karna (emit)
emitter.emit('order', 'Biryani'); // Order aya: Biryani

Common Errors & Debugging

Ye galtiyan mat karna!

❌
Error 1: Cannot find module 'express'Matlab: Express install nahi hua. Fix: npm install express run karo.
❌
Error 2: EADDRINUSE – Port already in useMatlab: Port 3000 par koi aur program chal raha hai. Fix: lsof -i :3000 se check karo ya port change karo.
⚠️
Galti: Callback HellNested callbacks ek dusre ke andar likhna — code unreadable ho jata hai. Fix: Promises ya async/await use karo!
⚠️
Galti: Blocking Code in Serverfs.readFileSync server mein mat karo — poora server ruk jata hai. Fix: Hamesha async methods use karo.
