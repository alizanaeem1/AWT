Express.js Kya Hai?

Jab hum sirf Node.js use karte hain to web app banana bohat mushkil ho jata hai — URL manually parse karna parta hai, HTTP methods khud handle karne parte hain, sab kuch ek hi jagah likhna parta hai. Express.js in tamam problems ka solution hai. Yeh ek fast aur simple framework hai jo Node.js ke upar bana hua hai.

Simple definition: Express.js ek Node.js framework hai jo web applications aur APIs banane ko asaan banata hai. Ise "de facto standard" kehte hain — matlab sabse zyada use hone wala Node.js framework.
⚡
Minimalist
Sirf zaroori cheezein deta hai — unnecessary complexity nahi. Node.js ki powers intact rehti hain.
🔓
Unopinionated
Koi specific pattern force nahi karta. MVC use karo ya kuch aur — choice tumhari hai.
🧩
Extensible
Middleware aur plugins se features add kar sakte ho jitne chahiye.
🏎️
Fast
V8 JavaScript engine pe build hai — performance top-notch hai.
Exam Tip 🎯 — "Unopinionated" ka matlab yeh hai ke Express tumhe koi specific architecture follow karne par majboor nahi karta. Yeh flexibility iska sabse bada feature hai.
02

Middleware — Darmiyani Kaam karne wala

Middleware wo functions hote hain jo request aane ke baad aur response bhejna se pehle kaam karte hain. Sochon jaise security guard — jab request andar aati hai, pehle guard check karta hai, phir aage bhejta hai.

Formula: Middleware ko teen cheezein milti hain: req (request), res (response), aur next (agle middleware ko call karne ka function).
Basic Middleware Example · app.js 
// Yeh middleware har request pe chalega
app.use((req, res, next) => {
  console.log('Request time:', Date.now());
  next(); // Zaroor call karo warna request ruk jayegi!
});
⚠️ Important: Agar next() call nahi kiya, toh request stuck ho jayegi aur koi response nahi milega. Yeh sabse common mistake hai!

Middleware ke Types

Type Use Kahan Hoti Hai Example
Application-level Puri app pe — app.use() se Logging, body parsing
Router-level Sirf ek router ke routes pe Auth check for /admin
Error-handling 4 parameters: (err, req, res, next) 500 error page
Built-in Express ke saath aati hai express.json()
Third-party NPM se install karo morgan, cors
Error Handling Middleware 
// Error middleware mein 4 parameters hote hain — yeh zaroor yaad rakho!
app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).send('Kuch toot gaya!');
});
🎯 Exam Key Point: Middleware ka order matter karta hai. Jo pehle app.use() mein likha, woh pehle chalega. Error handling middleware hamesha last mein likhi jaati hai.
03

Routing — Request Kahan Jaye?

Routing decide karta hai ke jab koi specific URL pe request aaye toh kaunsa code chalega. Jaise ek post office — har letter ka address dekh ke decide hota hai use kahan deliver karna hai.

Basic Routing Syntax 
// Formula: app.METHOD(PATH, HANDLER)

app.get('/', (req, res) => {
  res.send('Hello World!');
});

app.post('/users', (req, res) => {
  res.send('Naya user banao');
});

app.delete('/users/:id', (req, res) => {
  res.send(`User ${req.params.id} delete ho gaya`);
});

HTTP Methods — Kaam kya karte hain?

📥
GET
Data lena (Read). Jaise users ki list mangana ya koi ek user ka data dekhna.
📤
POST
Naya data submit karna (Create). Jaise naya account banana ya form bhejna.
✏️
PUT
Poora data replace karna (Update). Puri cheez badal dena.
🗑️
DELETE
Data hatana. Koi record ya resource permanently delete karna.
✅ PATCH vs PUT: PUT pura object replace karta hai jabke PATCH sirf kuch specific fields update karta hai. Interview mein yeh farq poochha jata hai!
04

Route Parameters — Dynamic URLs

Kabhi kabhi URL mein koi specific value aati hai — jaise user ka ID ya product ka number. Inhe route parameters se handle karte hain. Colon (:) se define hote hain aur req.params se access hote hain.

Route Parameters Example 
// URL: /users/123/books/456
app.get('/users/:userId/books/:bookId', (req, res) => {
  const userId = req.params.userId;   // '123'
  const bookId = req.params.bookId;   // '456'
  res.send(`User: ${userId}, Book: ${bookId}`);
});
📌 Yaad Rakho: Route parameter colon (:) se shuru hota hai URL mein, aur req.params.paramName se value milti hai. Multiple parameters ek hi URL mein ho sakte hain.
05

express.Router() — Modular Routing

Jab app bari hoti hai toh ek hi app.js mein sab kuch likhna messy ho jata hai. express.Router() se alag alag files mein routes organize kar sakte hain — jaise har department ka alag office hota hai.

Problem: App.js mein 100+ routes likhne se code unmanageable ho jata hai. Solution: Routes ko alag files mein divide karo, phir main app mein mount karo.
routes/users.js — Alag Router File 
const express = require('express');
const router = express.Router(); // Naya router banao

router.get('/', (req, res) => {
  res.send('Sare users');
});

router.get('/:id', (req, res) => {
  res.send(`User ID: ${req.params.id}`);
});

router.post('/', (req, res) => {
  res.send('Naya user banao');
});

module.exports = router; // Export karo
app.js — Router Mount Karna 
const express = require('express');
const app = express();

const usersRouter = require('./routes/users');

app.use(express.json());
app.use('/api/users', usersRouter); // Mount at /api/users

// Ab /api/users → users.js mein jayega
// /api/users/123 → router.get('/:id') handle karega

app.listen(3000);

Router ke Fayde

  • Organization: Related routes ek jagah rehte hain — dhundna asaan.
  • Maintainability: Ek feature change karna puri app ko affect nahi karta.
  • Reusability: Ek router multiple jagah use ho sakta hai.
  • Scalability: Team mein kaam karna asaan — har developer apne section pe kaam kare.

App Directory Structure

Ek professional Express app ka structure hamesha organized hota hai. Har cheez ki apni jagah hoti hai — controllers, models, routes, middleware sab alag folders mein.

Typical Express App Folder Structure 
my-app/
├── app.js              ← Main entry point
├── package.json
├── config/             ← Database settings
│   └── index.js
├── routes/             ← Route definitions
│   ├── users.js
│   └── products.js
├── controllers/        ← Business logic
│   ├── usersController.js
│   └── productsController.js
├── models/             ← Database schemas
│   ├── User.js
│   └── Product.js
├── middleware/         ← Custom middleware
│   ├── auth.js
│   └── logger.js
└── public/             ← Static files
FolderKya Hota Hai Yahan
app.jsExpress setup, global middleware, routes mount karna
routes/API endpoints define karna — controller se link karna
controllers/Request handle karna, business logic — model se baat karna
models/Database schema — data ka structure define karna
middleware/Reusable middleware functions
config/Environment variables, DB connection settings
07

Request Flow — Step by Step

Jab client koi request bhejta hai tab se response milne tak kya kya hota hai — yeh samajhna zaroor hai. Yeh sequence exam mein poochha ja sakta hai.

Example: GET /api/users/123

1
Client Request Bhejta Hai
Browser ya app se GET /api/users/123 request aati hai server ki taraf.
2
app.js Request Receive Karta Hai
Main entry point pe request pahunchti hai.
3
Global Middleware Chalti Hai
express.json(), logging middleware — request process hoti hai.
4
Path Match aur Router Forward
/api/users match hota hai — usersRouter ko forward ho jaati hai.
5
Router Middleware (Optional)
Authentication middleware check karta hai — authorized hai ya nahi?
6
Controller Function Call Hoti Hai
/:id match pe getUserById controller chalti hai.
7
Database Se Data
Controller, Model se poochha aur Model database se data lata hai.
8
Response Client Ko
Controller res.json() se response bhejta hai — client ko data milta hai!

08

📌 5 Key Points — Exam Se Pehle Yaad Karo

1️⃣
Express.js minimalist aur unopinionated framework hai — yeh Node.js ke upar build hota hai aur complex apps ko asaan banata hai. Koi specific pattern force nahi karta.
2️⃣
Middleware functions req, res, next receive karte hain — yeh request-response cycle ke beech chalte hain. next() call karna zaroor hai warna request stuck ho jati hai.
3️⃣
Routing decide karta hai ke kaunsa handler kaunse URL pe chalega — GET, POST, PUT, DELETE, PATCH sab alag purposes ke liye hain. Route parameters :paramName se define hote hain.
4️⃣
express.Router() se modular routing hoti hai — alag files mein routes define karo, phir main app mein mount karo. Yeh code ko maintainable aur scalable banata hai.
5️⃣
Separation of concerns — routes, controllers, models sab alag — professional apps mein har layer ka kaam alag hota hai. Isse debugging aur team collaboration asaan hoti hai.
