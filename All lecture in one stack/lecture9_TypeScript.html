Class Kya Hoti Hai?

Class ek blueprint hoti hai — jis se hum objects banate hain. Ek class ek baar likhte hain aur us se hazaron objects create kar sakte hain.

🚗
Car Blueprint
Class = Car ka design → jis se hazaron cars banti hain
🎓
Student Template
Class = Student ka structure → har student ek object hai
🏦
Bank Account
Class = Account ka design → deposit, withdraw methods ke saath
📌 Yaad Rakhne Wali Baat:
Class khud koi cheez nahi hai — yeh sirf design hai. Jab hum new keyword use karte hain, tabhi actual object banta hai.
📝

Class ka Syntax aur Structure

Class mein 3 main cheezein hoti hain: Properties, Constructor, aur Methods.

student.ts
class Student {
  // Properties — class ke variables
  name: string;
  age: number;

  // Constructor — object banate waqt chaltha hai
  constructor(name: string, age: number) {
    this.name = name;
    this.age = age;
  }

  // Method — class ke andar function
  display(): void {
    console.log(`Name: ${this.name}, Age: ${this.age}`);
  }
}

// Object banana
const s1 = new Student("Ali", 20);
s1.display(); // Output: Name: Ali, Age: 20
📦
Properties
Class ke andar variables — jaise name, age, balance
🔧
Constructor
Object banate waqt automatically chaltha hai — initial values set karta hai
⚡
Methods
Class ke andar functions — jaise display(), deposit(), withdraw()
🔐

Access Modifiers

Access Modifiers batate hain ke class ke properties/methods ko bahar se access kiya ja sakta hai ya nahi. TypeScript mein 3 modifiers hain:

public
Har jagah se access ho sakta hai — default hota hai
private
Sirf class ke andar — bahar se access nahi
protected
Class aur uske children mein — lekin bahar nahi
bankAccount.ts
class BankAccount {
  private balance: number; // sirf class ke andar accessible
  public owner: string;   // har jagah se accessible

  constructor(owner: string, balance: number) {
    this.owner = owner;
    this.balance = balance;
  }

  deposit(amount: number): void {
    this.balance += amount; // ✅ andar se access OK
  }
}

const acc = new BankAccount("Ali", 5000);
// acc.balance = 9999; ❌ Error! private hai
acc.deposit(1000); // ✅ method public hai
💡 Private kyun use karein?
Balance directly change nahi honi chahiye — sirf deposit() ya withdraw() se honi chahiye. Yahi Encapsulation hai — data ko protect karna!

extends Keyword

TypeScript mein inheritance ke liye extends keyword use hota hai. Child class parent ki sab cheezein le leti hai.

animal.ts
// Parent Class
class Animal {
  move(): void {
    console.log("Animal chal raha hai...");
  }
}

// Child Class — Animal ko extend kiya
class Dog extends Animal {
  bark(): void {
    console.log("Bhow bhow!"); // Dog ka apna method
  }
}

const d = new Dog();
d.move(); // ✅ Parent se mila — "Animal chal raha hai..."
d.bark(); // ✅ Dog ka apna — "Bhow bhow!"
📌 Important: Dog ko move() likhna nahi para — Animal se automatically mil gayi. Yahi inheritance ka beauty hai!
🔁

Method Overriding

Method Overriding matlab — child class parent ki method ko apne hisaab se badal deti hai. Naam same rehta hai lekin kaam alag hota hai.

override.ts
class Animal {
  speak(): void {
    console.log("Animal kuch kehta hai");
  }
}

class Cat extends Animal {
  // Override! Cat apna speak define karti hai
  speak(): void {
    console.log("Meow meow!");
  }
}

class Dog extends Animal {
  speak(): void {
    console.log("Bhow bhow!");
  }
}

new Cat().speak(); // "Meow meow!"
new Dog().speak(); // "Bhow bhow!"
🤔 Overriding kyun useful hai?
Sab animals speak() karte hain — lekin har animal alag tarah! Override se hum general behavior ko specific bana sakte hain. Yahi Polymorphism hai OOP mein!

Interface Kya Hai?

Interface ek contract hoti hai — jo batati hai ke class mein kya hona chahiye. Interface khud koi kaam nahi karti — sirf rules define karti hai.

📄
Job Contract
Contract batata hai — kya kaam karna hai. Employ karna employee ka kaam hai.
🔌
USB Port Standard
USB ka standard define hai — jo bhi device banao, us standard ko follow karna hoga
person.ts
// Interface — sirf structure, koi implementation nahi
interface Person {
  name: string;
  age: number;
}

// Class jo interface ko "implement" karti hai
class Student implements Person {
  name: string; // zaroori hai — interface ne kaha
  age: number; // zaroori hai — interface ne kaha

  constructor(name: string, age: number) {
    this.name = name;
    this.age = age;
  }
}
⚠️ Important Rule: Agar class ne interface implement ki aur koi property miss ki — TypeScript immediately Error dega! Interface ensure karta hai sab zaroori cheezein hain.
⚙️

implements — Interface with Methods

Interface mein methods bhi define kar sakte hain — lekin sirf signature, implementation nahi. Har class apna implementation karti hai.

shapes.ts
// Interface — area() ka sirf rule
interface Shape {
  area(): number;
}

// Circle apna area calculate karta hai
class Circle implements Shape {
  radius: number;
  constructor(r: number) { this.radius = r; }

  area(): number {
    return Math.PI * this.radius ** 2;
  }
}

// Rectangle apna area calculate karta hai
class Rectangle implements Shape {
  constructor(private w: number, private h: number) {}

  area(): number {
    return this.w * this.h;
  }
}

console.log(new Circle(5).area()); // 78.53...
console.log(new Rectangle(4,6).area()); // 24
🧩

Multiple Interfaces

TypeScript mein ek class ek saath kai interfaces implement kar sakti hai — yeh inheritance se alag aur powerful feature hai!

multiple.ts
interface A { a: number; }
interface B { b: number; }

// Ek class dono interfaces implement kar sakti hai
class Test implements A, B {
  a = 10; // Interface A ki requirement
  b = 20; // Interface B ki requirement
}
🔑 Key Difference:
Class sirf ek parent class se extend ho sakti hai — lekin kai interfaces implement kar sakti hai. Multiple inheritance ke liye interfaces use karo!

Combined Real-World Example

Teeno concepts — Interface + Class + Inheritance — ek saath dekho:

vehicles.ts
// Interface — rules define karta hai
interface VehicleRules {
  startEngine(): void;
  stop(): void;
}

// Class — interface implement karti hai
class Car implements VehicleRules {
  startEngine(): void {
    console.log("Car ka engine start hua — Vroom!");
  }
  stop(): void {
    console.log("Car ruk gayi");
  }
}

// Subclass — Car ko extend karti hai
class ElectricCar extends Car {
  startEngine(): void { // Override!
    console.log("Electric car silently start — zzz~");
  }
  chargeBattery(): void { // Electric ka apna method
    console.log("Battery charge ho rahi hai...");
  }
}

const tesla = new ElectricCar();
tesla.startEngine(); // "Electric car silently start"
tesla.stop(); // Car se mili — "Car ruk gayi"
tesla.chargeBattery(); // Electric ka apna method
📌 Summary:
VehicleRules (Interface) → rules define kiye
Car (Class) → interface implement kiya
ElectricCar (Subclass) → Car extend ki + override kiya
