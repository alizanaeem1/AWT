HTTP Server in Node.js

20 – 50 Minutes

💡
Core IdeaNode.js ka http module low-level server banata hai. Har request par hume 2 objects milte hain: req (IncomingMessage) aur res (ServerResponse).
📥REQ

req Object

req.method, req.url, req.headers — client ka request info. Request body bhi isme aata hai.

📤RES

res Object

res.statusCode, res.setHeader(), res.end() — client ko jawab dene ke liye.

server.js – Pehla Server! 
const http = require('http');

const server = http.createServer((req, res) => {
  // Response status set karo
  res.statusCode = 200;
  
  // Header set karo
  res.setHeader('Content-Type', 'text/plain; charset=utf-8');
  
  // Response bhejo aur connection band karo
  res.end('Mera pehla Node.js Server! 🎉\n');
});

// Port 3000 par sunna shuru karo
server.listen(3000, () => {
  console.log('Server chal raha hai: http://localhost:3000');
});
Status CodeMatlabKab Use Hota Hai

201 CreatedSuccessNaya resource bana
400 Bad RequestClient ErrorGalat data bheja
404 Not FoundClient ErrorPage/resource nahi mila
500 Server ErrorServer ErrorServer mein kuch gadbad

URL Module – Parse Karna

URL Ka Anatomy

⚠️
Zaroori Baat!req.url sirf path + query string hoti hai, puri URL nahi. Ise parse karne ke liye URL class use karo. Manual splitting (? aur & se karna) error-prone hai — encoding problems aate hain!
JavaScript – URL Parsing 
const { URL } = require('url');

// Server mein URL parse karna:
function parseUrl(req) {
  // Base URL zaroori hai kyunki req.url relative hoti hai
  const base = `http://${req.headers.host}`;
  return new URL(req.url, base);
}

// Example: /search?q=node&limit=10
const u = parseUrl(req);

console.log(u.pathname);                    // '/search'
console.log(u.searchParams.get('q'));     // 'node'
console.log(u.searchParams.get('limit')); // '10'
console.log(u.hash);                       // '#section' (client-side only)
❌ Galat Tarika (Manual Split)
const parts = req.url.split('?')
const query = parts[1].split('&')

❌ URL encoding handle nahi hoti
❌ Special characters break ho jate hain
❌ Error-prone code
✅ Sahi Tarika (URL Class)
const u = new URL(req.url, base)
u.searchParams.get('q')

✅ URL encoding automatic handle
✅ Clean, readable code
✅ Reliable aur standard

Routing — Bina Framework Ke

Method + URL = Handler

🗺️
Routing kya hai?Routing matlab — decide karo ke kaun si function chalegi based on HTTP method (GET/POST) aur URL path. Express automatically karta hai, lekin andar se yahi hota hai!
1

Routes Object Banao

Ek object mein sare routes define karo: key = "METHOD /path", value = handler function.

2

URL Parse Karo

Har request mein URL parse karo aur method + pathname se key banao.

3

Handler Dhoondo

Routes object mein key se matching handler function dhoondo.

4

404 Handle Karo

Agar koi route nahi mila, to 404 Not Found response bhejo.

JavaScript – Routing System 
const http = require('http');
const { URL } = require('url');

// Routes object — key: "METHOD /path"
const routes = {
  'GET /': (req, res) => {
    res.writeHead(200, { 'Content-Type': 'text/plain' });
    res.end('Home Page!\n');
  },
  'GET /api/time': (req, res) => {
    const data = { time: new Date().toISOString() };
    res.writeHead(200, { 'Content-Type': 'application/json' });
    res.end(JSON.stringify(data));
  }
};

function handler(req, res) {
  const u = new URL(req.url, `http://${req.headers.host}`);
  const key = `${req.method} ${u.pathname}`;
  const fn = routes[key];
  
  if (!fn) {
    res.writeHead(404, { 'Content-Type': 'text/plain' });
    return res.end('404 – Page Nahi Mila!\n');
  }
  fn(req, res, u);
}

http.createServer(handler).listen(3000);
console.log('http://localhost:3000 par chal raha hai');

File System (FS) Module

Files Padhna aur Likhna

⛔ Synchronous (Blocking)
Server rukta hai file padhne tak. Ek user ke liye poora server block ho jata hai! Server mein kabhi use mat karo.

fs.readFileSync('file.txt') ← BAD!
✅ Asynchronous (Non-Blocking)
Server doosre kaam karta rehta hai jab tak file read nahi hoti. Production mein yahi use karo!

fs.readFile() / fs.promises ← GOOD!
JavaScript – FS APIs 
const fs = require('fs/promises');  // Modern promise-based

// File padhna (async/await)
async function fileParho() {
  try {
    const data = await fs.readFile('data.txt', 'utf8');
    console.log('File ka content:', data);
  } catch (err) {
    console.error('File nahi mili!', err.message);
  }
}

// File likhna
async function fileLikho() {
  await fs.writeFile('output.txt', 'Mera Data!', 'utf8');
  console.log('File likh di!');
}

fileParho();
fileLikho();

Streams & pipe()

Large Files Efficiently Serve Karo

🌊
Stream kya hai?Stream data ko chhote chhote chunks mein process karta hai. Poori file RAM mein load karne ki zarurat nahi! Imagine karo: pipe se pani aata hai — tank bhar ke nahi laate, seedha flow karta hai.
📖READABLE

Readable Stream

Data ka source. Example: fs.createReadStream() file se data padhta hai chunk by chunk.

✍️WRITABLE

Writable Stream

Data ka destination. HTTP response ek writable stream hai — chunks receive karta hai.

🔧PIPE

pipe() Method

Readable ko Writable se connect karta hai. Backpressure automatic handle karta hai!

🛡️BACKPRESSURE

Backpressure

Network slow ho to pipe() data flow slow karta hai — memory overflow se bachata hai.

JavaScript – Stream + pipe() 
const fs = require('fs');

function fileBhejo(filePath, res) {
  // File ka stream banao — RAM mein poora load nahi hoga
  const fileStream = fs.createReadStream(filePath);

  // Error handle karo (file nahi mili, permission nahi)
  fileStream.on('error', (err) => {
    res.writeHead(404, { 'Content-Type': 'text/plain' });
    res.end('File nahi mili!\n');
  });

  // Response header set karo
  res.writeHead(200, { 'Content-Type': 'application/octet-stream' });
  
  // pipe() se file seedha response mein bhejo!
  fileStream.pipe(res); // Backpressure automatic!
}

Full Practical Examples

Complete Working Code

Example A: GET /api/echo?msg=hello — URL se query parameter leke JSON mein response do.

Example A – JSON API + URL Query 
const http = require('http');
const { URL } = require('url');

http.createServer((req, res) => {
  const u = new URL(req.url, `http://${req.headers.host}`);
  
  // Sirf GET /api/echo handle karo
  if (req.method === 'GET' && u.pathname === '/api/echo') {
    const msg = u.searchParams.get('msg') ?? '';
    const payload = { ok: true, msg };
    res.writeHead(200, { 'Content-Type': 'application/json' });
    return res.end(JSON.stringify(payload));
  }
  
  // Koi aur route nahi mila
  res.writeHead(404);
  res.end('Not Found\n');
}).listen(3000);

Example B: Static files serve karo public/ folder se — streams use karke, security ke saath.

Example B – Static File Server 
const http = require('http');
const fs   = require('fs');
const path = require('path');
const { URL } = require('url');

const publicDir = path.join(__dirname, 'public');

// Content-Type map
function getContentType(filePath) {
  const ext = path.extname(filePath).toLowerCase();
  return ({
    '.html': 'text/html; charset=utf-8',
    '.css':  'text/css; charset=utf-8',
    '.js':   'application/javascript',
    '.json': 'application/json',
    '.png':  'image/png',
    '.jpg':  'image/jpeg'
  })[ext] || 'application/octet-stream';
}

// Security: path traversal rokna
function safeJoin(base, target) {
  const finalPath = path.normalize(path.join(base, target));
  if (!finalPath.startsWith(base)) return null; // Attack block!
  return finalPath;
}

http.createServer((req, res) => {
  const u = new URL(req.url, `http://${req.headers.host}`);
  let reqPath = decodeURIComponent(u.pathname);
  if (reqPath === '/') reqPath = '/index.html';

  const filePath = safeJoin(publicDir, reqPath);
  if (!filePath) { res.writeHead(400); return res.end('Bad Request'); }

  const stream = fs.createReadStream(filePath);
  stream.on('error', () => { res.writeHead(404); res.end('File nahi mili'); });
  res.writeHead(200, { 'Content-Type': getContentType(filePath) });
  stream.pipe(res);
}).listen(3000);

Security — Path Traversal

Zaroori Hai Jaanna!

🚨
Path Traversal Attack kya hai?Hacker URL mein /../secret.txt likh ke public folder se bahar ki files access karne ki koshish karta hai. Agar check nahi kia to server ke private files leak ho sakte hain!
1

URL Decode Karo

decodeURIComponent() se URL ke encoded characters decode karo — attacker %2F use kar sakta hai.

2

path.normalize() Use Karo

/../.. patterns normalize ho jaate hain — ../.. ka jo bhi effect hai wo remove hota hai.

3

startsWith() Check Karo

Final path verify karo ke wo public directory ke andar hai ya nahi. Bahar hai to 400/403 do.

Common Errors & Debugging

Ye galtiyan mat karna!

❌
Galti 1: res.end() bhool gayeAgar response end nahi kia, connection hang rehta hai, browser loading karta rehta hai. Hamesha res.end() ya pipe() use karo!
❌
Galti 2: Galat Content-TypeJSON bhej rahe ho lekin Content-Type text/html set hai — browser confuse ho jata hai. Hamesha sahi Content-Type set karo.
⚠️
Galti 3: readFile for Large FilesBadi file ko readFile se padhna memory overload kar sakta hai. Large files ke liye hamesha streams use karo!
⚠️
Galti 4: Error Handling Nahifs aur stream errors handle nahi karna server crash kar sakta hai. Hamesha try/catch ya .on('error') use karo.
