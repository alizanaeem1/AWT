React Unit Testing — Sahi Mindset

RTL: user jaisa dekhe / kare — wahi assert karo

✅
High-value tests:

Empty / loading / error states • click, type, submit • conditional UI • getByRole, getByLabelText — accessibility bhi improve hoti hai.

⛔
Aksar skip karo:

Private functions, internal state structure, CSS pixels — visual regression tools ke baghair.

👤
RTL rule:

“Query like a user” — getByRole, getByLabelText, findByText (async ke liye).

Example: Cart mein “Add” dabao → item count barhe — yahi behaviour test karo, na ke internal useState setter.

📊

Unit vs Integration vs E2E

Testing pyramid yaad rakho

TypeScopeToolingExample
UnitEk component / functionJest + RTLButton click → label update
IntegrationComponents + API moduleJest + RTL + MSW/mockForm submit → success message
E2EPoora browser appCypress / PlaywrightLogin → dashboard

🔺 Pyramid

Niche: zyada unit tests. Beech: kuch integration. Top: kam E2E — slow lekin high confidence.

⚡ Speed

Unit sab se tez; E2E sab se slow — balance zaroori.

🎯

Practical Patterns

Events, forms, async, mocks

1

Events & state

Click → UI change; type → value; submit → validation messages.

2

Async UI

Loading spinner → data aaye → error path bhi cover karo.

3

Mocking

api.js module mock karna — har jagah fetch mock karna avoid jahan behtar ho. MSW-style approach conceptual.

❌
Common mistakes:

Async content ke liye getByText — use findBy... ya waitFor. Async actions wrap karna bhoolna. Internal state test karna. Over-mocking — tests meaningless.

📝
Exam angle:

“API UI kaise test?” → mock network, loading → success/error UI assert, calls verify.

UsersList example: pehle “Loading…” dikhe, phir list render ho — yehi sequence assert karo.

Code Examples

RTL + Express essentials

RTL — smoke test (concept)
import { render, screen } from '@testing-library/react';
import userEvent from '@testing-library/user-event';

test('button dabane pe greeting dikhe', async () => {
  render(<Welcome />);
  await userEvent.click(screen.getByRole('button', { name: /hello/i }));
  expect(screen.getByText(/assalam/i)).toBeInTheDocument();
});
Async — findBy
// async list ke liye getByText ki bajaye:
const item = await screen.findByText(/loaded row/i);
Express — layers remind
