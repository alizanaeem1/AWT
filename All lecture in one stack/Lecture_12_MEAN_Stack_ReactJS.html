REST API Design (Coding Se Pehle)

Resources + methods + status codes — contract clear hona chahiye

🔑
Important:

Agar frontend aur backend ke JSON keys match nahi karti, app “broken” lagegi chahe dono sides ka code sahi ho.

Method Route (example) Kaam Typical status
GET /api/tasks Saari tasks list 200 OK
POST /api/tasks Nayi task create 201 Created
PUT/PATCH /api/tasks/:id Update (e.g. done flag) 200 OK
DELETE /api/tasks/:id Task delete 200/204

Naming: /api/tasks achha hai, /api/getTasks jaisa verb-heavy REST style mein avoid karo. Status codes: 400 bad request, 404 not found, 500 server error.

📝
Exam tip:

REST mapping table (method + route + purpose + status code) likhna seekho — theory mein aksar aata hai.

Hooks = Lifecycle — useEffect + Fetch

Class lifecycle ki jagah hooks

  • useEffect(() => { ... }, []) — pehli render ke baad ek dafa (componentDidMount jaisa idea).
  • Dependency array matter karti hai: [] vs [tasks] vs [query] — galat array = infinite loop ya stale data.
  • Flow: loading set → API call → state set → error handle — UI mein loading/error dikhana mat bhoolna.
🐞
Demo bug idea:

Teacher jaan boojh ke dependency miss kare — infinite re-fetch loop dikhao; phir theek karo.

🔮

Virtual DOM & Rendering

Har change pe poora page reload nahi

React har baar poora real DOM redraw nahi karta — pehle Virtual DOM banata hai, diff (reconciliation) karta hai, phir jo zaroori hai sirf wahi patch karta hai.

🔑
Lists + keys:

List se item hatao jab key stable nahi hoti — UI glitch / wrong row update ho sakta hai. State immutability bhi reconciliation ke liye important hai.

Code Examples

API contract + React fetch pattern

JSON — POST /api/tasks 
// Request body (example)
{ "title": "Buy milk", "done": false }

// Response (example)
{ "_id": "...", "title": "Buy milk", "done": false }
React — fetch on load (concept) 
import { useState, useEffect } from 'react';

function TaskApp() {
  const [tasks, setTasks] = useState([]);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    fetch('http://localhost:5000/api/tasks')
      .then(r => r.json())
      .then(data => { setTasks(data); setLoading(false); })
      .catch(() => setLoading(false));
  }, []);

  // ... JSX: loading ? spinner : TaskList
}
Express — essentials snippet 
const express = require('express');
const cors = require('cors');
const app = express();

app.use(cors());
app.use(express.json()); // zaroori — warna req.body undefined

Common Errors & Debugging

Full-stack integration — yahi masle zyada aate hain

🚫

CORS blocked

Frontend alag origin — server pe cors() enable karo.

🔗

Route typo

/api/task vs /api/tasks — 404 mile to URL match dekho.

🔤

id vs _id

Mongo _id bhejta hai — frontend agar id expect kare to mismatch.

📦

req.body undefined

express.json() missing — sabse common backend bug.

🔄

Stale UI after POST

POST ke baad list dubara fetch ya state update — warna purana data dikhega.

∞

useEffect loop

Galat dependencies — infinite requests; devtools Network tab dekho.

CORS kyun? List mein key na ho to kya? useEffect infinite loop kyun?
