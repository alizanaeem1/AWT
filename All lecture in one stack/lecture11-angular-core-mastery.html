Part B — 10 se 30 Minute

Templating — Data ko UI se Jodo! 📝

Templates se aap values display karte ho aur user actions pe react karte ho. Chaar tarike hain data bind karne ke — sab yaad karo!

1️⃣
Interpolation
Component ki value text ki tarah dikhao. Double curly braces!
// Component:
title = 'Mera Store';
// Template:
<h1>{{ title }}</h1>
2️⃣
Property Binding
DOM element ki property set karo. Square brackets use karo!
imgUrl = '/logo.png';
// Template:
<img [src]="imgUrl">
<button [disabled]="isDisabled">
3️⃣
Event Binding
User actions pakro — click, input, submit. Round brackets!
addToCart(item: any) {
  /* cart mein add karo */
}
// Template:
<button (click)="addToCart(item)">
4️⃣
Two-Way Binding
Dono taraf sync — input se component, component se input. FormsModule zaroor!
searchText = '';
// Template:
<input [(ngModel)]="searchText">
<p>{{ searchText }}</p>

Template Reference Variable

template-ref.html 
<!-- #emailInput ek reference variable hai -->
<input #emailInput type="email">
<button (click)="check(emailInput.value)">Check</button>
<!-- Is se directly DOM element access kar sakte hain -->

    Directives — DOM ko Control Karo! 🎛️

    Directives DOM ki structure ya appearance ko modify karne ke instructions hain. Do qisam hain — Structural aur Attribute!

    🏗️ Structural Directives — DOM badlte hain

    DirectiveKya Karta HaiUse Case
    *ngIfElement add/remove karta haiConditional UI
    *ngForElement repeat karta haiLists / Tables
    *ngSwitchSwitch-case renderingMultiple states
    structural.html 
    <!-- ngIf: show/hide -->
<div *ngIf="isLoggedIn">
  Profile dikhao
</div>
<button *ngIf="!isLoggedIn">Login</button>

<!-- ngFor: list -->

    🎨 Attribute Directives — Style/Behavior badlte hain

    DirectiveKya Karta Hai

    [ngStyle]Inline styles apply karta hai
    ngModelForm control binding karta hai
    attribute.html 
    <!-- ngClass: conditional class -->
<div [ngClass]="{'active': isActive, 'disabled': !isActive}">
  Status
</div>

<!-- ngStyle: inline style -->

  Styled Text
</p>
    ✅
    Zaroori Baat!*ngIf element ko DOM se REMOVE karta hai — sirf hide nahi karta. Iska performance aur event handlers pe asar padta hai. Yaad rakhna!
    🎯
    Exam Tip — Common Question!Structural vs Attribute Directives: Structural DOM ki structure badlte hain (*ngIf, *ngFor). Attribute sirf appearance/behavior badlte hain (ngClass, ngStyle). * (asterisk) structural ki pehchan hai!
    Part D — 50 se 60 Minute

    Pipes — Data Format Karo! 🔧

    Pipes template mein data ko format/transform karte hain — original data bilkul nahi badlta! Sirf display ke liye hain.

    Built-in Pipes — Ready-Made!

    {{ 1200 | currency:'USD' }}→$1,200.00currency pipe
    {{ today | date:'short' }}→4/22/26, 3:00 PMdate pipe
    {{ 'ahmed' | uppercase }}→AHMEDuppercase pipe
    {{ 3.14159 | number:'1.2-2' }}→3.14number pipe
    {{ obs$ | async }}→auto subscribe!async pipe

    Custom Pipe — Khud Banao!

    truncate.pipe.ts 
    @Pipe({ name: 'truncate' })
export class TruncatePipe implements PipeTransform {
  transform(value: string, limit = 50): string {
    return value.length > limit
      ? value.substring(0, limit) + '...'
      : value;
  }
}
// Template mein use:
// {{ longText | truncate:30 }}
    ⚠️
    Common Galti!Pipe mein heavy computation mat karo — ye bar bar run hota hai! Aur pipe business logic ke liye nahi, sirf display formatting ke liye hai.
    🎯
    Exam Tip!"Pure pipe" sirf tab run karta hai jab input change ho. "Impure pipe" baar baar run karta hai — ye expensive ho sakta hai performance ke liye!

    Dependency Injection + Services 💉

    DI ka matlab hai Angular khud service objects banata aur deta hai — component ko manually nahi banani parti. Ye Angular ka superpower hai!

    Component Request
    →
    Angular Injector
    →
    Provider Dhoondhe
    →
    Instance Return
    product.service.ts 
    @Injectable({
  providedIn: 'root'
  // Poori app mein ek hi instance
  // = Singleton pattern
})
export class ProductService {
  private products = [
    { id: 1, name: 'Laptop' },
    { id: 2, name: 'Phone' }
  ];

  getProducts() {
    return this.products;
  }
}
    home.component.ts 
    @Component({ selector: 'app-home', ... })
export class HomeComponent {

  products: any[] = [];

  // Constructor mein inject karo
  constructor(
    private ps: ProductService
  ) {}

  ngOnInit() {
    // Service use karo!
    this.products = this.ps.getProducts();
  }
}

    Services Kyun? — Fayde!

    🧹
    Component Saaf Rahe
    Component sirf UI dekhta hai. API calls, caching, auth — sab service mein!
    ♻️
    Logic Reuse Karo
    Ek service likho — poori app mein use karo. Baar baar same code likhne ki zaroorat nahi!
    🧪
    Testing Aasaan
    Service ko alag test kar sakte ho. Mock service inject karo — component test simple ho jaata hai!
    Provider ScopeMatlabEffect
    providedIn: 'root'Root levelPoori app mein ek hi instance (Singleton)
    component providers: []Component levelHar component ka alag instance — unexpected ho sakta hai!
    ⚠️
    Common Galtiyan!HttpClient code seedha component mein likhna — ye galat hai, service mein rakho! Aur component-level provider accidentally multiple instances bana sakta hai — dhyan rakhna!

    Observables — Reactive Data Streams! 📡

    Observables async data ki streams represent karte hain — API responses, events, route changes. Angular HttpClient Observable return karta hai!

    Route Change
    →
    ActivatedRoute Observable
    →
    HttpClient Observable
    →
    async pipe render
    product.service.ts 
    @Injectable({ providedIn: 'root' })
export class ProductService {
  constructor(private http: HttpClient) {}

  getProducts(): Observable<Product[]> {
    return this.http.get<Product[]>(
      '/api/products'
    );
  }

  getById(id: number): Observable<Product> {
    return this.http.get<Product>(
      `/api/products/${id}`
    );
  }
}
    products.component.ts + html 
    // Method 1: Manual subscribe
ngOnInit() {
  this.ps.getProducts()
    .subscribe(data => {
      this.products = data;
    });
}

// Method 2: async pipe (BETTER!)
products$ = this.ps.getProducts();

// Template mein:
// <li *ngFor="let p of products$ | async">
//   {{ p.name }}</li>
// async pipe khud subscribe/unsubscribe karta hai!

    RxJS Operators — Powerful Tools!

    🗺️
    map()
    Data transform karo. Jaise response se sirf names nikalo!
    🔍
    filter()
    Conditions lagao. Sirf active products dikhao!
    ⚡
    switchMap()
    Pehla request cancel karo, nayi request chalao. Search-as-you-type ke liye perfect!
    ⚠️
    Memory Leak Alert!subscribe() karo aur cleanup na karo to memory leak hoti hai! async pipe use karo — wo khud unsubscribe karta hai. Ya takeUntil / Subscription cleanup pattern use karo!
    💡
    Real-World ExampleSearch-as-you-type: user type kare, debounce lage, switchMap purani request cancel kare, nayi API call ho — sab Observables se possible!
    Common Galtiyan

    Debugging Clinic 🐛

    Ye galtiyan sabse zyada hoti hain — pehle se yaad karo to kabhi phase nahi khao ge!

    🔴
    subscribe() everywhere — cleanup nahiLong-lived streams mein subscribe karo aur unsubscribe na karo — memory leak hogi! async pipe ya takeUntil use karo.
    🔴
    Pipe mein business logicPipe sirf display formatting ke liye hai. Agar logic complex hai — Service mein rakho, pipe mein nahi!
    🔴
    ** route pehle likh diWildcard route HAMESHA last mein honi chahiye — warna sab routes usi pe match ho jaenge!
    🔴
    HttpClient directly component meinAPI calls service mein honi chahiye. Component mein directly HttpClient use karna bad practice hai!
    🔴
    router-outlet bhool gayeBina <router-outlet> ke routes kaam nahi karenge. App component ke template mein lagana zaroor hai!
    💡
    Pro TipBrowser DevTools Console kholo — Angular ke error messages bohat descriptive hote hain. Pehle console check karo!
