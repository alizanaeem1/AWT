Part 1 — Classes

Blueprint → objects

📌
Class kya hai?

Class blueprint hai objects banane ka — jaise car ka blueprint → bohot cars; student template → multiple student instances.

TypeScript
class Student {
  name: string;
  age: number;

  constructor(name: string, age: number) {
    this.name = name;
    this.age = age;
  }

  display(): void {
    console.log(`Name: ${this.name}, Age: ${this.age}`);
  }
}

Properties

Class ke andar variables — state hold karti hain.

Constructor

Object initialize — values set karta hai.

Methods

Class ke andar functions — behaviour.

🏦
Bank account example (notes)

Properties: account number, balance • Methods: deposit(), withdraw().

🔐

Access Modifiers

Encapsulation — TypeScript

Modifier Meaning
public Har jagah accessible (default behaviour bhi often public treat)
private Sirf class ke andar — bahar direct access nahi
protected Class + subclasses ke liye
private balance
class BankAccount {
  private balance: number;

  constructor(balance: number) {
    this.balance = balance;
  }

  deposit(amount: number) {
    this.balance += amount;
  }
}
💬
Discussion prompt:

Agar fields private kar dein to kya hota hai? — direct tampering kam, controlled methods zyada (encapsulation).

Part 2 — Inheritance

extends — reuse + specialization

Inheritance = ek class doosri class ki properties/methods dobara use karti hai. Real-life intuition: child inherits traits from parents — code me base → derived.

extends
class Animal {
  move(): void {
    console.log("Moving...");
  }
}

class Dog extends Animal {
  bark(): void {
    console.log("Barking...");
  }
}
🔑
Keyword: extends — class inheritance ke liye.

Hierarchy example: Parent Vehicle → children Car, Bike.

Method overriding

override speak()
class Animal {
  speak(): void {
    console.log("Animal speaks");
  }
}

class Cat extends Animal {
  speak(): void {
    console.log("Cat meows");
  }
}
♻️

Code reuse

Common logic base class me — specific child me.

🧹

Cleaner structure

Hierarchy se domain model readable.

🛠️

Maintenance

Base change careful — ripple effect samajh ke.

Part 3 — Interfaces

Contract — implements

📌
Interface kya hai?

Ye contract define karta hai jo class follow kare — jaise job description / USB standard: “kya shapes honi chahiye”, implementation details interface me nahi hoti.

implements Person
interface Person {
  name: string;
  age: number;
}

class Student implements Person {
  name: string;
  age: number;

  constructor(name: string, age: number) {
    this.name = name;
    this.age = age;
  }
}
✅
Type match:

Interface Person ke age: number ke sath class properties bhi same shape me hon — warna TS error.

Shape + Circle
interface Shape {
  area(): number;
}

class Circle implements Shape {
  radius: number;

  constructor(radius: number) {
    this.radius = radius;
  }

  area(): number {
    return Math.PI * this.radius * this.radius;
  }
}

Multiple interfaces

implements A, B
interface A { a: number; }
interface B { b: number; }

class Test implements A, B {
  a = 10;
  b = 20;
}
❓
Discussion:

Interfaces par “constructor” directly interface body me waise hi nahi jaise class — TS me pattern alag hai; contract fields/method signatures pe focus.

⚖️

Part 4 — Classes vs Interfaces

PDF comparison table

Feature Class Interface
Implementation Haan — methods ki body ho sakti hai Nahi — structure / signatures
Object creation (new) Class se instance Interface se directly object nahi
Constructor Haan Nahi (interface me constructor pattern alag discussion)
Inheritance-style extension extends single class Interfaces can extends multiple interfaces
Multiple “inheritance” for one class Single superclass (TS) — multiple implements Multiple interfaces implement ho sakti hain
💡
When to use?

Class jab runtime behaviour + state + implementation chahiye. Interface jab shared shape/contract chahiye (APIs, testing mocks, multiple classes same contract).

🌍
Combined example (notes)

interface VehicleRules → class Car implements VehicleRules → class ElectricCar extends Car.
