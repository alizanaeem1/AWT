Fetch API

Built-in — manual JSON + error discipline

1

fetch(url)

Call karo — promise return.

2

response.ok

404/500 par fetch reject NAHI karta — hamesha check karo.

3

response.json()

await — parse body.

4

State / dispatch

Data UI ya Redux ko do.

✅
Headers:

POST ke liye Content-Type: application/json + body stringify.

🔁
StrictMode (dev):

Effects do dafa chal sakte — duplicate fetch rokne ke liye guard / AbortController cancel.

⚠️
Mistakes:

await json() bhoolna • POST headers missing • unmount ke baad setState (cleanup / abort).

Redux + React — Core

Predictable global state

  • Store — single source of truth.
  • Reducer — pure function (state, action) → newState. API calls yahan NAHI.
  • Action — plain object { type, payload }.
  • React: <Provider store={store}> • useSelector read • useDispatch send.
📌
Immutability:

Reducer mein state mutate mat karo — nayi copy return karo.

❌
Mistakes:

Reducer mein mutation • galat key useSelector mein • wrong action.type.

🔗

Redux Thunk — Async

dispatch function — API yahan

1

FETCH_START

loading true, error clear.

2

API call

fetch/axios thunk ke andar.

3

SUCCESS / FAIL

data ya error dispatch — reducer update.

🎯
Exam line:

Thunk middleware lets you dispatch functions receiving (dispatch, getState) — async actions.

Login example: dispatch start → API → dispatch success(user/token).

Code Examples

Simplified snippets

Fetch GET + ok check
const res = await fetch('/api/products');
if (!res.ok) throw new Error('HTTP ' + res.status);
const data = await res.json();
Axios
const { data } = await axios.get('/api/products');
// data === parsed JSON
Thunk shape (concept)
const loadProducts = () => async (dispatch) => {
  dispatch({ type: 'PRODUCTS_PENDING' });
  try {
    const data = await api.getProducts();
    dispatch({ type: 'PRODUCTS_SUCCESS', payload: data });
  } catch (e) {
    dispatch({ type: 'PRODUCTS_FAIL', error: e.message });
  }
};
