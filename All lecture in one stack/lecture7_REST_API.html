API Kya Hoti Hai?

API ka matlab hai Application Programming Interface. Yeh ek set of rules hota hai jo different software systems ko ek dosre se baat karne deta hai.

🍕 Real-life Example – Foodpanda ya Careem:
Jab tum app use karte ho, mobile app server se API ke through baat karta hai.
App request bhejta hai: "Show available restaurants"
Server JSON format mein response bhejta hai.
📱
Client (Frontend)
Woh jo request karta hai — mobile app ya browser
🔗
API (Bridge)
Waiter ki tarah — client aur server ke beech connection
🖥️
Server (Backend)
Actual data store karta hai aur process karta hai
🔄

REST Kya Hai?

REST ka matlab hai Representational State Transfer. Yeh ek architectural style hai jo web applications design karne ke liye use hoti hai. Isme HTTP protocol use hota hai.

REST ke 6 Principles

🔌
Client-Server
Frontend aur Backend alag alag hote hain
📭
Stateless
Server kuch yaad nahi rakhta — har request apna data khud laata hai
⚡
Cacheable
Responses cache ho sakte hain performance ke liye
📐
Uniform Interface
Standard URLs aur HTTP methods use hoti hain
🏗️
Layered System
Security layer, database layer — alag alag ho sakti hain
📄
Code on Demand
Optional — server code bhi bhej sakta hai
💡 Stateless ka example – ATM:
Har baar ATM pe jaate ho, apna card dobara daalna parta hai. Server kuch save nahi karta!
📡

HTTP Methods

REST APIs mein 4 main HTTP methods use hote hain — ek ek kaam ke liye alag method hota hai.

Method Kaam Kya Hai? Example
GET Data retrieve karo (padhna) Products ki list dekho
POST Naya data create karo User register karo
PUT Existing data update karo Profile update karo
DELETE Data delete karo Account delete karo
🤔 Facebook Login mein kaunsa method?
Login ke liye POST use hota hai — kyunke credentials (password) request body mein securely bhejte hain. GET method URL mein data show karta hai jo unsafe hota hai!

CRUD Operations Code

CRUD = Create, Read, Update, Delete — ye 4 main operations hain jo har API mein hote hain.

GET – Saare Students Dekho

app.get('/api/students', (req, res) => {
  res.json(students); // saara list bhejo
});

GET – ID se Ek Student Dekho

app.get('/api/students/:id', (req, res) => {
  const student = students.find(s => s.id == req.params.id);
  if (!student) {
    return res.status(404).json({ message: "Student nahi mila" });
  }
  res.json(student);
});

POST – Naya Student Add Karo

app.post('/api/students', (req, res) => {
  const newStudent = { id: students.length + 1, ...req.body };
  students.push(newStudent);
  res.status(201).json(newStudent); // 201 = Created
});

PUT – Student Update Karo

app.put('/api/students/:id', (req, res) => {
  const student = students.find(s => s.id == req.params.id);
  if (!student) return res.status(404).json({ message: "Nahi mila" });
  student.name = req.body.name;
  student.email = req.body.email;
  res.json(student);
});

DELETE – Student Delete Karo

app.delete('/api/students/:id', (req, res) => {
  students = students.filter(s => s.id != req.params.id);
  res.json({ message: "Student delete ho gaya" });
});

CORS – Cross-Origin Resource Sharing

CORS ek security mechanism hai jo browser enforce karta hai. Yeh control karta hai ke kaunsa domain server se data maang sakta hai.

🏠 Hostel Example:
Tum Hostel A mein rehte ho. Sirf Hostel A ke log andar aa sakte hain.
Agar Hostel B ka koi aana chahe to permission chahiye — yehi CORS hai!

CORS Headers Server pe Set Hote Hain

# Server in headers bhejta hai
Access-Control-Allow-Origin: https://example.com
Access-Control-Allow-Methods: GET, POST
Access-Control-Allow-Headers: Content-Type

Best Practices

❌ KABHI MAT KARO
Sensitive APIs ke liye * (sabko allow)
Banking/finance APIs pe unrestricted CORS
✅ SAHI TARIKA
Sirf trusted domains allow karo
Methods restrict karo
Server side token validate karo
⚠️

XSS aur DDoS Attacks

🎭 XSS – Cross-Site Scripting
Attacker malicious script trusted website mein inject karta hai
  • Cookie chori hoti hai
  • Session hijack hoti hai
  • User malicious site pe redirect hota hai
🌊 DDoS – Distributed Denial of Service
Hazaron computers ek saath server ko requests se flood karte hain
  • Server crash ho jata hai
  • Real users access nahi kar paate
  • Service down ho jaati hai
🏪 DDoS Real Example (2016):
Dyn company pe attack hua jisse Twitter, Netflix, aur GitHub sab down ho gaye the — yehi DDoS ka impact hota hai!

XSS Example – Comment Box Attack

# Attacker comment box mein ye likhta hai:
<script>alert("Hacked!")</script>

# Agar sanitize nahi kiya to har user page pe ye script chalti hai!

Protection Techniques

🧹
Input Sanitize Karo
User input ko always clean karo — HTML/scripts strip karo
⏱️
Rate Limiting
Ek IP se zyada requests block karo
🔒
HTTPOnly Cookies
JavaScript cookies access nahi kar sake
🌐
CDN + Load Balancer
DDoS traffic distribute karo, server protect karo

Authentication vs Authorization

🔐 AUTHENTICATION (AuthN)
TUM KAUN HO?
Username + Password check karna
OTP verify karna
Fingerprint scan
🔑 AUTHORIZATION (AuthZ)
TUMHARA ACCESS KAHAN HAI?
Student admin panel nahi dekh sakta
Teacher marks upload kar sakta hai
Admin sabko delete kar sakta hai
🏫 University Example:
ID Card Check = Authentication (tum kaun ho?)
Lab Access = Authorization (kya tum lab use kar sakte ho?)

JWT – JSON Web Token

JWT ek compact token format hai jo REST APIs mein authentication ke liye use hota hai. RFC 7519 standard follow karta hai.

JWT Structure – 3 Parts Hain

# Format: Header.Payload.Signature
eyJhbGciOiJIUzI1NiJ9.eyJ1c2VySWQiOjF9.SflKxwRJSMeKKF2QT4fwpMeJf
📋
Header
Algorithm batata hai — jaise HS256
📦
Payload
User data hota hai — jaise user ID, role
✍️
Signature
Verify karta hai token genuine hai ya nahi

JWT Flow – Login se Access Tak

1
User Login Karta Hai
Username + password bhejta hai
2
Server Verify Karta Hai
Credentials database se match karta hai
3
JWT Token Generate Hota Hai
Server token bana ke client ko deta hai
4
Client Token Store Karta Hai
LocalStorage ya cookie mein save hota hai
5
Har Request mein Token Bhejta Hai
Authorization: Bearer <token> header mein
6
Server Token Verify Karta Hai
Sahi hai to access milta hai, nahi to 401 error
🔒

Session Handling in REST

REST stateless hota hai — matlab server kuch yaad nahi rakhta. To session kaise handle karein?

❌ OLD METHOD – Session-Based
Server session store karta tha
Client ke paas sirf Session ID hoti thi
Scale karna mushkil tha
Microservices mein kaam nahi karta
✅ NEW METHOD – Token-Based (JWT)
Server sirf token verify karta hai
Client token store karta hai
Stateless — scale easily hota hai
Mobile apps ke saath bhi kaam karta hai

Secure Session Practices

🔐
HTTPS Use Karo
Sab traffic encrypted honi chahiye
⏰
Short Token Expiry
Token jald expire ho — 15 min ya 1 hour
🔄
Refresh Tokens
Expired token renew karne ke liye
🚫
Compromised Token Revoke
Stolen token ko blacklist karo
