HTTP Basics — Bunyadi Baatein

HTTP protocol ka core samajhna

HTTP matlab HyperText Transfer Protocol. Ye ek request-response system hai. Client (browser) request bhejta hai, server response deta hai.

HTTP Methods — Kya karna chahte ho?

Method Matlab Example
GET Data mangna (read karna) Users ki list mangna
POST Naya data banana (create) Naya user register karna
PUT / PATCH Data update karna Profile update karna
DELETE Data delete karna Account delete karna

Status Codes — Server kya jawab de raha hai?

✅

200 OK

Sab theek hai. Request successful rahi.

🆕

201 Created

Naya resource ban gaya. POST ke baad aata hai.

❓

404 Not Found

Jo cheez dhundh rahe ho, woh mili nahi.

⚠️

400 Bad Request

Client ne galat request bheji. Data check karo.

🔴

500 Internal Error

Server side par kuch problem hai.

🎯
Exam Tip

Status codes yaad rakho: 2xx = success, 4xx = client error (teri galti), 5xx = server error (server ki galti). Ye exam mein zaroor aata hai!

req aur res Objects

Node.js ke do important dost

Jab koi request aati hai, Node.js hume do objects deta hai. Inhe samajhna bahut zaroori hai:

📥

req (Request)

IncomingMessage object hai. Isme hote hain:
• req.method — GET ya POST
• req.url — path + query
• req.headers — request headers

📤

res (Response)

ServerResponse object hai. Isse response bhejo:
• res.statusCode — 200, 404 etc
• res.setHeader() — headers set karo
• res.end() — response bhejo

JavaScript — server.js 
// Sabse simple HTTP server
const http = require('http');

const server = http.createServer((req, res) => {
  // req se information lein
  console.log(req.method);  // 'GET'
  console.log(req.url);     // '/search?q=node'

  // res se jawab dein
  res.statusCode = 200;
  res.setHeader('Content-Type', 'text/plain');
  res.end('Hello from Node!');
});

server.listen(3000, () => {
  console.log('Server chal raha hai: http://localhost:3000');
});
⚠️
Common Mistake

res.end() call karna mat bhoolo! Agar ye nahi likhoge to response kabhi nahi jayega aur browser hang hoga.

Routing — Traffic Control

Alag URLs pe alag kaam karna

Routing matlab: kaunse URL pe kaunsa kaam karna hai. Jaise ek traffic policeman decide karta hai ke kaunsi gaari kidhar jaye.

Hum ek smart pattern use karenge jahan method + pathname ek key banate hain:

JavaScript — Routing Pattern 
const http = require('http');
const { URL } = require('url');

// Routes ka object — method + path => function
const routes = {
  'GET /': (req, res) => {
    res.writeHead(200, { 'Content-Type': 'text/plain' });
    res.end('Home Page\n');
  },
  
  'GET /api/time': (req, res) => {
    const data = { now: new Date().toISOString() };
    res.writeHead(200, { 'Content-Type': 'application/json' });
    res.end(JSON.stringify(data));
  }
};

function handler(req, res) {
  const u = new URL(req.url, `http://${req.headers.host}`);
  const key = `${req.method} ${u.pathname}`;
  
  const fn = routes[key];
  if (!fn) {
    res.writeHead(404, { 'Content-Type': 'text/plain' });
    return res.end('404 Not Found\n');
  }
  fn(req, res, u); // Sahi function call karo
}

http.createServer(handler).listen(3000);
💡
Teaching Note

Ye Express.js ka simple version hai! Express bhi isi tarah kaam karta hai lekin aur features ke sath. Ye samajh lo to Express automatically samjh aa jayega.

URL Module — Link Tod Phod

URL ko properly parse karna

URL kya hota hai? Jaise: http://localhost:3000/search?q=node&limit=10

Isko alag alag parts mein todna padta hai. URL class ye kaam safe tareeqe se karta hai.

URL ke Parts

Property Example Value Matlab
pathname /search Route path — kahan jana chahte ho
searchParams q=node&limit=10 Query string — extra information
hostname localhost Server ka naam
port 3000 Server ka port
hash #section1 Page ke andar link (browser side)
JavaScript — URL Parsing 
const { URL } = require('url');

// req.url sirf '/search?q=node' hota hai (pura nahi)
// Isliye base dena padta hai
function parseUrl(req) {
  const base = `http://${req.headers.host}`;
  return new URL(req.url, base);
}

// Example:
const u = parseUrl(req);

console.log(u.pathname);                    // '/search'
console.log(u.searchParams.get('q'));      // 'node'
console.log(u.searchParams.get('limit')); // '10'
⚠️
String Splitting Mat Karo!

req.url.split('?')[1] jaisi cheezein mat karo. Agar URL mein special characters ya encoding ho to ye toot jata hai. URL class sab handle karta hai.

File System (FS) Module

Files padhna aur likhna

Node.js ka fs module hume file system se interact karne deta hai. Files padhna, likhna, delete karna — sab kuch.

3 Tareeqe Files Padhne Ke

1

Synchronous (Blocking) — Ruk ke kaam karo

Puri file padh ke hi aage jao. Server ko rok deta hai. Production mein mat use karo!

2

Async Callback — Kaam karo, batao baad mein

File padhne ka kaam shuru karo, baad mein callback function call hoga. Server baaki requests handle karta raha.

3

Promise / Async-Await — Modern tareeqa ✨

Sabse clean code. await fs.readFile() likh do aur kaam ho jayega. Ye server-side ke liye best practice hai.

JavaScript — FS Comparison 
// ❌ 1. Synchronous — Server rok leta hai
const fs = require('fs');
const data = fs.readFileSync('file.txt', 'utf8');

// ✅ 2. Async Callback
fs.readFile('file.txt', 'utf8', (err, data) => {
  if (err) throw err;
  console.log(data);
});

// ✅✅ 3. Promise / Async-Await (BEST)
const fs = require('fs/promises');

async function readFile() {
  const data = await fs.readFile('file.txt', 'utf8');
  console.log(data);
}
🎯
Exam Tip — Golden Rule

Server mein hamesha async APIs use karo. Sync APIs event loop ko block kar deti hain. Ek user ki wajah se baaki sab wait karenge!

Streams aur pipe() — Nali ka System

Badi files efficiently serve karo

Sochो: agar tumhein 1GB ki video file serve karni ho, kya poori file RAM mein load karoge? Nahi!

Streams ek nali ki tarah kaam karte hain. Data thoda thoda aata hai, thoda thoda client ko bhejte ho. RAM kam lagti hai, speed zyada hoti hai.

pipe() — Jadu ki Nali

pipe() ek readable stream ko directly writable stream se jod deta hai. HTTP response ek writable stream hai, to file stream seedha response mein jata hai!

JavaScript — File Streaming 
const fs = require('fs');

function streamFileToResponse(filePath, res) {
  // File ka readable stream banao
  const fileStream = fs.createReadStream(filePath);
  
  // Agar file nahi mili to error handle karo
  fileStream.on('error', (err) => {
    res.writeHead(404, { 'Content-Type': 'text/plain' });
    res.end('File nahi mili!\n');
  });
  
  res.writeHead(200, { 'Content-Type': 'application/octet-stream' });
  
  // pipe() = file → response (automatic chunking + backpressure)
  fileStream.pipe(res);
}
🔵

Readable Stream

File, database query — ye data deta hai. fs.createReadStream() se banao.

🟢

Writable Stream

HTTP response, file write — ye data leta hai. res object writable stream hai.

🔮

Backpressure

Network slow ho to pipe() data ka flow khud control karta hai. Memory safe rehti hai.

Security — Khatraa Khatraa!

Path Traversal attack se bachao

Sochو: agar koi URL type kare /../../../etc/passwd to kya hoga? Agar hum seedha file serve kar dein to private files bhi expose ho sakti hain!

Ise Path Traversal Attack kehte hain. Isse bachne ka tareeqa:

JavaScript — Safe Path Check 
const path = require('path');

// Public folder ka path
const publicDir = path.join(__dirname, 'public');

function safeJoin(base, target) {
  // normalize() '../..' jaisi cheezein resolve karta hai
  const targetPath = path.normalize(path.join(base, target));
  
  // Check: kya path public folder ke andar hai?
  if (!targetPath.startsWith(base)) {
    return null; // Khatre ki baat! null return karo
  }
  
  return targetPath; // Safe path
}

// Use karte waqt:
const filePath = safeJoin(publicDir, reqPath);
if (!filePath) {
  res.writeHead(400);
  return res.end('Bad Request');
}
🛡️
Security Rule #1

Kabhi bhi user ke diye gaye path ko directly file system par use mat karo. Hamesha path.normalize() karo aur check karo ke path allowed folder ke andar hai.

Code Examples — Poore Server

Real duniya ke examples

Example A — JSON API with Query Params

GET /api/echo?msg=hello — ye endpoint query param leta hai aur JSON mein wapas deta hai.

JavaScript — Echo API 
const http = require('http');
const { URL } = require('url');

http.createServer((req, res) => {
  const u = new URL(req.url, `http://${req.headers.host}`);
  
  // Sirf GET /api/echo handle karo
  if (req.method === 'GET' && u.pathname === '/api/echo') {
    const msg = u.searchParams.get('msg') ?? '';
    const payload = { ok: true, msg: msg };
    
    res.writeHead(200, { 'Content-Type': 'application/json' });
    return res.end(JSON.stringify(payload));
  }
  
  // Baaki sab ke liye 404
  res.writeHead(404);
  res.end('Not Found');
  
}).listen(3000);

// Test: http://localhost:3000/api/echo?msg=hello
// Response: {"ok":true,"msg":"hello"}

Example B — Static File Server (Complete)

Ye poora static file server hai jo public/ folder se files serve karta hai — security ke sath!

JavaScript — Static File Server 
const http = require('http');
const fs   = require('fs');
const path = require('path');
const { URL } = require('url');

const publicDir = path.join(__dirname, 'public');

// Extension se Content-Type milta hai
function getContentType(filePath) {
  const ext = path.extname(filePath).toLowerCase();
  const types = {
    '.html': 'text/html',
    '.css':  'text/css',
    '.js':   'application/javascript',
    '.png':  'image/png',
    '.jpg':  'image/jpeg'
  };
  return types[ext] || 'application/octet-stream';
}

http.createServer((req, res) => {
  const u = new URL(req.url, `http://${req.headers.host}`);
  let reqPath = decodeURIComponent(u.pathname);
  
  // / ko index.html par redirect karo
  if (reqPath === '/') reqPath = '/index.html';
  
  // Safe path check
  const full = path.normalize(path.join(publicDir, reqPath));
  if (!full.startsWith(publicDir)) {
    res.writeHead(400); 
    return res.end('Bad Request');
  }
  
  // File stream karo
  const stream = fs.createReadStream(full);
  stream.on('error', () => {
    res.writeHead(404);
    res.end('File nahi mili');
  });
  res.writeHead(200, { 'Content-Type': getContentType(full) });
  stream.pipe(res);
  
}).listen(3000);
