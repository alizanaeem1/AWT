Middleware, Routing & Routers

// Part 01

Express.js kya hai? 🚀

Node.js ke upar banaya gaya ek minimal aur flexible web framework — jo server banana bahut aasaan bana deta hai.

Socho ke tum ek restaurant chala rahe ho. Customers aate hain (browser), order dete hain (HTTP request), kitchen mein khaana banta hai (server logic), aur waiter deliver karta hai (response). Express.js is poore system ka manager hai!

🍽️ Restaurant Analogy

Express.js ko samajhne ka sabse aasaan tarika:

👤 Customer→Client/Browser
📝 Order→HTTP Request
👨‍🍳 Kitchen→Server Logic
🛎️ Waiter→Middleware
👔 Manager→Express.js
⚡

Minimal Framework

Express bahut chhota aur fast hai. Zyada unnecessary features nahi — sirf jo chahiye wo deta hai.

🔌

Node.js ke upar

Express sirf Node.js ka wrapper hai. Node ki power use karta hai lekin kaam aasaan kar deta hai.

🌐

APIs & Web Servers

REST APIs banana ho ya web server — Express dono ke liye perfect hai.

💡
Tip: Express ko install karne ke liye terminal mein likho: npm install express
javascript — Basic Express App 
// Step 1: Express ko import karo
const express = require('express');

// Step 2: App object banao
const app = express();

// Step 3: Route define karo
app.get('/', (req, res) => {
  res.send("Welcome to Express!");
});

// Step 4: Server start karo port 3000 par
app.listen(3000, () => {
  console.log("Server running on port 3000 ✅");
});

Middleware kya hota hai? 🔄

Middleware wo function hai jo request receive hone ke baad aur response bhejne se pehle execute hota hai.

Middleware ek security guard ki tarah hai. Office mein ghusne se pehle aapko guard ke paas se guzarna padta hai — woh check karta hai, stamp lagata hai, phir aage bhejta hai (next()). Agar koi problem ho to wahan hi rok leta hai.

✈️ Airport Analogy

Airport pe jaane ka process bilkul middleware jaisa hai:

👤 Passenger arrives→Request aai
🛂 Passport check→Middleware 1 (Auth)
🧳 Baggage scan→Middleware 2 (Validation)
🚪 Boarding gate→Route Handler
💡
Yaad rakho: Middleware mein teen parameters hote hain: req, res, aur next. Agar next() call nahi karoge to request aage nahi jayegi!
javascript — Middleware Example 
// Middleware Function — Har request par chalega
app.use((req, res, next) => {
  console.log(`Request aayi: ${req.method} ${req.url}`);
  next(); // Aage bhejo
});

// Auth Middleware — Sirf logged-in users ke liye
const checkAuth = (req, res, next) => {
  if (req.headers.token === 'secret123') {
    next(); // Token theek hai, aage jao
  } else {
    res.status(401).send("Unauthorized! Login karo pehle.");
  }
};

// Is route par auth middleware apply karein
app.get('/dashboard', checkAuth, (req, res) => {
  res.send("Dashboard — Sirf aap ke liye!");
});

Middleware ke Types

🌍

Application-Level

app.use() se define hota hai. Poori application par apply hota hai — har request se pehle chalega.

📂

Router-Level

Sirf ek specific router ke andar kaam karta hai. Alag module ke liye alag middleware use kar sakte ho.

📦

Built-in

Express ke saath aata hai. express.json() JSON parse karta hai. express.static() files serve karta hai.

🚨

Error-Handling

4 parameters hote hain: err, req, res, next. Koi bhi error aaye to yahi pakadta hai.

javascript — Error Handling Middleware 
// Error handler — 4 parameters zaroor likho!
app.use((err, req, res, next) => {
  console.error(err.message);
  res.status(500).send("Kuch toot gaya! 😅");
});
⚠️
Warning: Middleware ka order bahut important hai! Express code ko upar se neeche padhta hai. Agar auth middleware baad mein likha to security kaam nahi karegi.
TypeSyntaxUse Case
Application-Levelapp.use(fn)Logging, global auth
Router-Levelrouter.use(fn)Module-specific middleware
Built-in JSONapp.use(express.json())POST body parse karna
Built-in Staticapp.use(express.static())CSS/JS files serve karna
Error Handlerapp.use(err,req,res,next)Errors pakadna

Routing kya hoti hai? 🗺️

Routing se hum define karte hain ke kisi specific URL par request aane par kaun sa code chalega.

Socho Google Maps — alag alag destinations ke liye alag alag routes hote hain. Express mein bhi URL alag hogi to alag function chalega:

  • /home → Home page dikha
  • /about → About page dikha
  • /users → Users ki list do
javascript — Route Methods 
// GET — Data mangna (e.g., products ki list)
app.get('/products', (req, res) => {
  res.send("Yeh sab products hain");
});

// POST — Naya data bhejta (e.g., user register)
app.post('/register', (req, res) => {
  res.send("User register ho gaya!");
});

// PUT — Purana data update karna
app.put('/user/:id', (req, res) => {
  res.send(`User ${req.params.id} update hua`);
});

// DELETE — Data hatana
app.delete('/user/:id', (req, res) => {
  res.send("User delete ho gaya");
});

Route Parameters

Kabhi kabhi URL mein dynamic value chahiye hoti hai — jaise /user/5 mein 5 user ka ID hai. Iske liye :paramName use karte hain:

javascript — Route Parameters 
// :id ek dynamic parameter hai
app.get('/user/:id', (req, res) => {
  const userId = req.params.id; // URL se ID nikalo
  res.send(`User ID hai: ${userId}`);
});

// URL: /user/42  →  Output: User ID hai: 42
// URL: /user/99  →  Output: User ID hai: 99

📌
Exam Tip: GET, POST, PUT, DELETE — yeh sab HTTP methods CRUD operations ke saath match karte hain: Create → POST, Read → GET, Update → PUT, Delete → DELETE.

Express Router 📁

Jab routes zyada ho jayein to Router use karo — routes ko alag files mein organize karo aur code clean rakho.

Socho university mein alag alag departments hain:

🏫 University Analogy

Har department apna kaam khud karta hai — routes ki tarah:

📋 Admission Office→/admissions
💰 Accounts→/accounts
📝 Exams→/exams

Agar saare routes ek hi app.js mein likho to file messy ho jati hai. Router se har module ki alag file hoti hai!

javascript — routes/user.js 
// routes/user.js — User se related routes
const express = require('express');
const router = express.Router(); // Router object banao

// GET /users — Sab users
router.get('/', (req, res) => {
  res.send("Sab users ki list");
});

// GET /users/5 — Single user
router.get('/:id', (req, res) => {
  res.send(`User ${req.params.id}`);
});

module.exports = router; // Export karo
javascript — app.js (Main File) 
// app.js — Main application file
const express = require('express');
const app = express();

// User routes import karo
const userRoutes = require('./routes/user');
const productRoutes = require('./routes/product');

// Use karo — /users prefix ke saath
app.use('/users', userRoutes);
app.use('/products', productRoutes);

// Ab kaam karte hain:
// GET /users      → All users
// GET /users/10   → Single user (id=10)
// GET /products   → All products

app.listen(3000);
💡
Faida: Router se code clean rehta hai, team mein kaam baant sakte hain, aur debugging aasaan ho jati hai. 50 routes ek file mein — nightmare! Alag alag files mein — clean code! ✅

Large App Structure 🧱

Professional Express apps ek specific folder structure follow karte hain — MVC pattern ke mutabiq.

Jaise ek acha ghar banane ke liye drawing pehle banate hain, waise hi app ka structure pehle plan karte hain:

project/ │ ├── app.js ← Main entry point ├── package.json ← Dependencies │ ├── routes/ ← Sab routes yahan │ ├── user.js │ ├── product.js │ └── order.js │ ├── middleware/ ← Custom middleware │ ├── auth.js │ └── logger.js │ ├── controllers/ ← Business logic │ ├── userController.js │ └── productController.js │ ├── models/ ← Database schema │ ├── User.js │ └── Product.js │ └── config/ ← Configuration files └── db.js

MVC Pattern kya hai?

🗄️

Model (M)

Database se related code. User ka data kahan store hoga, schema kya hoga — yeh sab model mein hota hai.

🎨

View (V)

Jo user ko dikhta hai — frontend. EJS, React, ya HTML pages. Data ko visually present karna.

⚙️

Controller (C)

Business logic. Model se data lo, process karo, aur View ko do. Middleware aur routes ke darmiyan bridge.

javascript — Controller Example 
// controllers/userController.js
const userController = {

  // Sab users fetch karo
  getUsers: (req, res) => {
    // Database se users lao (yahan hardcoded)
    const users = [{ id: 1, name: 'Ahmed' }, { id: 2, name: 'Sara' }];
    res.json(users);
  },

  // Single user fetch karo
  getUser: (req, res) => {
    res.json({ id: req.params.id, name: 'Ahmed' });
  }

};

module.exports = userController;
⚠️
Middleware Order yaad rakho: Express mein middleware upar se neeche execute hota hai. Agar Logger pehle chahiye aur Auth baad mein, to Logger pehle likho!
javascript — Correct Middleware Order 
// ✅ SAHI ORDER:
app.use(logger);       // Pehle log karo
app.use(auth);         // Phir auth check karo
app.use('/users', userRoutes); // Routes at the end

// ❌ GALAT ORDER:
// app.use('/users', userRoutes);  ← Route pehle
// app.use(auth);                  ← Auth baad mein = koi fayda nahi!
