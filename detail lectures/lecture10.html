Part A · 0–10 Min

Angular Kya Hai? 🤔

Jab website bahut bari ho jaye — aur code sanbhalna mushkil ho jaye — tab Angular kaam aata hai. Yeh ek complete framework hai, sirf library nahi.

💡
Seedhi Baat!
Library vs Framework ka farq samjho: Library ko tum call karte ho (jese jQuery), lekin Framework tumhe call karta hai. Angular mein hum uske rules follow karte hain — yahi Angular ko powerful banata hai!
🏢
Enterprise-Level Apps
Angular badi badi companies use karti hain — HR portals, banking frontends, admin dashboards. Kyunki yeh structured hai, teams mein kaam karna asaan ho jata hai.
Real World Use
📝
TypeScript-First
Angular pure TypeScript mein likha jata hai — jo JavaScript ka "super version" hai. Isse errors jaldi milti hain aur code clean rehta hai. Types define karo, bugs khatam karo!
Language
🔧
Full Toolkit
Angular sirf UI nahi deta — routing, forms, HTTP, testing, animations sab built-in hain. Alag alag libraries dhundne ki zaroorat nahi. Ek package, sab kuch!
All-in-One
⚡
Component-Based UI
Har cheez chhoti chhoti "bricks" se bani hoti hai jinhein components kehte hain. Jese ghar mein alag kamre hote hain — yahan har kamra ek component hai!
Architecture
✅
Yaad Rakho!
Angular = MEAN stack ka "A" hai! MongoDB + Express + Angular + Node.js. Frontend ke liye Angular use hota hai jo browser mein chalta hai.
Part B · 10–25 Min

Angular ka Architecture 🏗️

Poori Angular app ka ek structure hota hai — samjho jese ek building ki blueprint. Is chain ko yaad karo: Module → Component → Template → DOM

Building Block Kya Karta Hai? Real-Life Example File
NgModule Components ko organize karta hai, configuration store karta hai School ki building — sab departments yahan hain app.module.ts
Component UI block — logic + view + style ek saath School mein ek classroom app.component.ts
Template HTML view jahan data display hota hai Classroom ki whiteboard app.component.html
Decorator Class ko batata hai "tu kya hai" — metadata deta hai Name tag jaise batata hai "Main Teacher hoon" @Component
Angular CLI App banana, serve karna, components generate karna Workshop ka toolbox ng new / ng serve
🎯
Exam Tip!
Exam mein pucha jaye "How does Angular render UI?" — to jawab yeh hoga: Component class + Template + Data Binding (decorators se drive hota hai). Yeh teen cheezein hamesha milke kaam karti hain!
Part C · 25–45 Min

Angular App Kaise Banate Hain? 💻

Ab practical karte hain! Angular CLI ka use karke ek naya app create karo. Sirf teen commands — aur tumhara app chal jayega!

  1. CLI Install Karo — Pehli baar sirf ek baar: npm install -g @angular/cli
  2. Naya App Banao — Jahan chahte ho wahan folder mein: ng new awt-angular-demo
  3. Folder mein jao — cd awt-angular-demo
  4. App Run Karo — Browser khud khul jayega: ng serve -o
  5. Folder Structure Dekho — src/app mein jao, wahan sara code hoga
TYPESCRIPT · app.component.ts 
// Yeh Angular ka basic component hai
import { Component } from '@angular/core';

@Component({
  selector: 'app-root',       // HTML mein yeh tag use hoga
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent {
  title = 'Hello AWT!';   // Yeh property hai
}
HTML · app.component.html 
<!-- Template mein data show karte hain -->
<div class="container">
  <h1>{{ title }}</h1>  <!-- Interpolation -->
  <p>Angular seekhna shuru!</p>
</div>
✅
Pro Tip!
ng serve wala server "hot reload" karta hai — matlab jab bhi tum HTML ya TS file save karo, browser khud refresh ho jata hai. Code likho, instantly result dekho!
Part D · 45–65 Min

NgModule — Angular ka Organizer 📦

Module Angular ko batata hai ki teri app mein kya kya hai. Jese library mein books ka catalog hota hai — module bhi waise hi Angular ka catalog hai.

TYPESCRIPT · app.module.ts 
import { NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { FormsModule } from '@angular/forms';
import { AppComponent } from './app.component';

@NgModule({
  declarations: [   // "Mere apne components"
    AppComponent,
    UserCardComponent
  ],
  imports: [        // "Jo modules mujhe chahiye"
    BrowserModule,
    FormsModule      // [(ngModel)] ke liye zaroori!
  ],
  providers: [],    // Services (DI ke liye)
  bootstrap: [AppComponent]  // Starting point
})
export class AppModule { }
Property Matlab Example Yahan Hota Hai
declarations "Yeh mere components hain" — jo is module ke andar hain AppComponent, UserCardComponent Sirf is module mein
imports "Yeh cheezein mujhe chahiye" — doosre modules FormsModule, RouterModule Doosri jagah se lata hai
providers "Yeh services inject karunga" HttpClient, AuthService Module ya component
bootstrap "Is component se start karo" AppComponent (sirf root module mein) Sirf AppModule mein
⚠️
Common Galtiyan!
1. Component ko imports mein daalna — nahi! Woh declarations mein aayega.
2. [(ngModel)] use karo aur FormsModule import na karo — Error aayega: "Can't bind to ngModel"!
Part E · 65–90 Min

Components — UI ke Building Blocks 🧩

Component Angular ki sabse chhoti aur important unit hai. Ek component mein teen cheezein hoti hain: Logic (TS) + View (HTML) + Style (CSS). Samjho jese ek brick — in bricks se hi poori app banti hai!

TYPESCRIPT · user-card.component.ts 
import { Component, Input } from '@angular/core';

@Component({
  selector: 'app-user-card',  // HTML tag: <app-user-card>
  template: `
    <div class="card">
      <h2>{{ userName }}</h2>
      <p>{{ userRole }}</p>
      <button (click)="toggleFollow()">
        {{ isFollowed ? 'Following ✓' : 'Follow' }}
      </button>
    </div>
  `,
  styles: ['.card { padding: 16px; border: 1px solid #ccc; }']
})
export class UserCardComponent {
  @Input() userName: string = 'Ali Ahmed';
  @Input() userRole: string = 'Student';
  isFollowed: boolean = false;

  toggleFollow() {
    this.isFollowed = !this.isFollowed;
  }
}
🏷️
selector
Yeh component ka HTML tag naam hai. selector: 'app-user-card' matlab kisi bhi HTML file mein <app-user-card> likh do — component wahan render ho jayega!
📋
templateUrl / template
templateUrl alag HTML file point karta hai. Chhoti cheezein template mein seedha likh sakte ho (backtick string mein).
🎨
styleUrls / styles
Har component ki apni CSS hoti hai — aur woh sirf us component tak limited hoti hai. Doosra component affect nahi hoga! Yahi "component encapsulation" hai.
📥
@Input() aur @Output()
@Input() se parent component data bhejta hai child ko. @Output() se child parent ko event bhejta hai. Jese messages opar neeche jaate hain!
🎯
Exam Tip!
Chain yaad karo: selector → HTML mein use karo → woh component ka template render hota hai. Yeh teen steps exam mein zaroor likhna!
TERMINAL · Component Generate Karna 
# Angular CLI khud files bana deta hai!
ng generate component user-card

# Ya short form mein:
ng g c user-card

# Yeh 4 files banata hai:
# user-card.component.ts
# user-card.component.html
# user-card.component.css
# user-card.component.spec.ts (testing)
Part F · 90–110 Min

Data Binding — Jaan Angular ki 🔗

Data Binding woh "bridge" hai jo TypeScript code ko HTML template se jodta hai. Jab TypeScript mein data badlega — HTML khud update ho jayega. Yahi Angular ki real power hai!

Type 1
Interpolation
📤 TypeScript → HTML (sirf dikhana)
{{ userName }}
Double curly braces mein variable ka naam likho — woh value HTML mein show ho jayegi. Jese template mein window hai — TypeScript ki value bahar dikhai deti hai.
Type 2
Property Binding
📤 TypeScript → DOM Property
[disabled]="isDisabled"
Square brackets use hote hain DOM property set karne ke liye. Jese button ko dynamically disable karna — agar isDisabled=true hai to button gray ho jayega!
Type 3
Event Binding
📥 DOM Event → TypeScript Method
(click)="save()"
Round brackets se events handle hote hain. Button click hoga to TypeScript mein save() method call hogi. Jese alarm bajao — watchman aaye!
Type 4
Two-Way Binding
🔄 TypeScript ↔ HTML dono taraf
[(ngModel)]="userName"
Dono taraf kaam karta hai — user input change kare to TypeScript update hoga, TypeScript change kare to HTML update hoga. Forms ke liye best hai! FormsModule import karna zaroori hai.
HTML · Charon Binding Ek Saath 
<!-- 1. Interpolation: Value directly dikhao -->
<h1>Student: {{ studentName }}</h1>

<!-- 2. Property Binding: DOM property set karo -->
<button [disabled]="isSubmitting">Submit</button>
<img [src]="profileImageUrl" [alt]="studentName">

<!-- 3. Event Binding: User action par method call karo -->
<button (click)="submitForm()">Save</button>
<input (input)="onTyping($event)">

<!-- 4. Two-Way Binding: Real-time sync -->
<input [(ngModel)]="studentName" placeholder="Naam likho">
<p>Live preview: {{ studentName }}</p>
⚠️
Common Binding Mistakes!
1. Event binding mein square brackets: [click]="save()" ❌ → (click)="save()" ✅
2. Interpolation attribute mein: src="{{url}}" ❌ → [src]="url" ✅
3. ngModel ke liye FormsModule bhool gaye → "Can't bind to ngModel" error!
