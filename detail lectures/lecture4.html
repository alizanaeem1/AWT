01

Express.js kya hai? Kyun use karein?

Framework ki zaroorat — beginner-friendly explanation

Socho aap ek bada ghar bana rahe ho. Node.js aapko sirf bricks deta hai (raw materials). Lekin Express.js ek complete toolkit hai — hammer, nails, plans sab kuch. Isse kaam kaafi aasaan ho jaata hai!

⚡
Minimalist hai
Express bahut chhota aur light hai. Woh sirf zaroori cheezein deta hai, Node.js ki power ko chhupaata nahi. Aap khud decide karte ho baaki kya add karna hai.
🔓
Unopinionated hai
Express aap pe koi rule nahi thopata — koi specific database use karo, koi specific pattern follow karo. Poori azadi hai developer ko!
🚀
Fast & Extensible
V8 JavaScript engine pe build hai, toh speed excellent hai. Aur thousands of middleware packages NPM pe available hain — har kaam ke liye kuch na kuch milta hai.

Node.js vs Express — Fark kya hai?

Feature Node.js (raw) Express.js
HTTP Server bananaManually sab likhna padta haiapp.listen() se ho jaata hai
RoutingURL manually parse karoapp.get(), app.post() simple hai
Body ParsingKhud handle karoexpress.json() built-in hai
MiddlewareKoi system nahiPowerful middleware pipeline
Code lengthZyada linesKam aur clean code
💡
TIP
Express ko "wrapper" samjho Node.js ka. Wohi kaam karta hai, lekin kaafi asaan aur organized tarike se!
02

Middleware — Request ka Safar

Har request ek pipeline se guzarti hai — woh pipeline middleware hai

Middleware ko ek security guard ki tarah samjho jo office building ke bahar khada hai. Har aane wala banda (request) pehle usi ke paas se guzarta hai. Guard check karta hai — ID hai? Authorized hai? Phir andhar jaane deta hai. Agar nahi? Wapas bhej deta hai. Express ka middleware bilkul aise kaam karta hai!

Basic Structure

JavaScript 
// Middleware function ka basic structure
app.use((req, res, next) => {
  // req  = client ki request ka object
  // res  = server ki response ka object
  // next = agla middleware call karne ka function

  console.log('Waqt hai:', Date.now());
  next(); // ZAROOR likho, warna request ruk jaayegi!
});
⚠️
WARNING — IMPORTANT!
Agar middleware mein next() nahi likha, toh request wahan ruk jaayegi aur client ko koi response nahi milega. Browser hang ho jaayega!

Types of Middleware

🌐
Application-level
app.use() se define hota hai. Sab routes pe apply hota hai, ya specific path pe. Global settings ke liye best.
📁
Router-level
router.use() se define hota hai. Sirf us specific router ke routes pe apply hota hai. Modular apps ke liye best.
🚨
Error-handling
4 parameters hote hain: (err, req, res, next). Sab ke baad define karo. Jab koi error aaye to yeh pakadta hai.
📦
Built-in
Express ke saath already included. express.json(), express.static(), express.urlencoded() — extra install nahi chahiye.
🛒
Third-party
NPM se install karo. Jaise morgan (logging ke liye), cors (cross-origin ke liye). Community ka bana hua.

Real Code Examples

JavaScript — Middleware Examples 
// 1. Built-in: JSON parsing enable karo
app.use(express.json());

// 2. Custom: Har request ka log raho
app.use((req, res, next) => {
  console.log(`[${new Date().toISOString()}] ${req.method} ${req.url}`);
  next();
});

// 3. Third-party: morgan logger
const morgan = require('morgan');
app.use(morgan('combined'));

// 4. Error-handling middleware (hamesha last mein)
app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).send('Kuch toot gaya! 😅');
});
🎓
EXAM TIP
Middleware ka order bahut important hai! Jo pehle app.use() se define kiya, woh pehle execute hoga. Error-handling middleware hamesha sab ke baad likhna chahiye.
03

Routing — Sahi Raasta Dikhao

URL + HTTP Method = Sahi Handler tak pahuncho

Routing ek traffic policeman ki tarah hai. Jab koi request aata hai, Express dekhta hai — "Yeh kaunse raaste pe aaya hai? Kya karna chahta hai?" Uske mutabiq sahi handler pe bhej deta hai.

Basic Syntax

JavaScript — Routing Syntax 
// app.METHOD(PATH, HANDLER)

// GET — Data lena (Read)
app.get('/', (req, res) => {
  res.send('Hello World!');
});

// POST — Data bhejna (Create)
app.post('/users', (req, res) => {
  res.status(201).json({ message: 'User ban gaya!' });
});

// PUT — Data update karna (Update)
app.put('/users/:id', (req, res) => {
  res.send(`User ${req.params.id} update ho gaya!`);
});

// DELETE — Data hatana (Delete)
app.delete('/users/:id', (req, res) => {
  res.send(`User ${req.params.id} delete ho gaya!`);
});

HTTP Methods — Simple Explanation

MethodKaam kya haiReal life example
GETData parhna / lenaNews website kholna
POSTNaya data bhejnaForm submit karna
PUTPoora data replace karnaProfile completely update karna
PATCHThodi si update karnaSirf name change karna
DELETEData hatanaAccount delete karna

Route Parameters — Dynamic URLs

Kabhi kabhi URL mein koi number ya ID hoti hai jo change hoti rehti hai. Jaise /users/123 ya /products/456. In dynamic values ko pakadne ke liye route parameters use hote hain — colon (:) se define karte hain.

JavaScript — Route Parameters 
// :userId aur :bookId dynamic hain
app.get('/users/:userId/books/:bookId', (req, res) => {
  const { userId, bookId } = req.params;

  // /users/123/books/456 request aaya toh:
  // userId = "123", bookId = "456"

  res.send(`User: ${userId}, Book: ${bookId}`);
});

// Multiple handlers chain kar sakte ho
const checkAuth = (req, res, next) => {
  /* authentication check */
  next();
};

app.get('/profile', checkAuth, (req, res) => {
  res.send('User ka profile data');
});
ℹ️
YAAD RAKHO
Route parameters req.params mein milte hain. Query strings jaise ?name=Ali req.query mein milte hain. POST body ka data req.body mein milta hai (express.json() ke baad).
04

express.Router() — Modular Banao

Badi app ko chhotay chhotay hisson mein todna

Socho ek hospital hai. Sab doctors ek hi kamre mein baith jayein toh kya hoga? Chaos! Isliye alag alag departments hote hain — cardiology, orthopedics, etc. express.Router() bhi aise hi kaam karta hai — related routes ko alag file mein rakhta hai.

Problem: App cluttered ho jaati hai

JavaScript — Bura tarika (sab ek jagah) 
// app.js — sab routes ek jagah... messy!
app.get('/users', ...);
app.post('/users', ...);
app.get('/products', ...);
app.post('/products', ...);
app.get('/orders', ...);
// ... saikdon lines... mushkil hai manage karna!

Solution: Router se alag files banao

1
Router file banao (routes/users.js)
Users ke saare routes alag file mein likho
↓
2
module.exports karo
Router ko export karo taake dusri file use kar sake
↓
3
app.js mein mount karo
app.use('/api/users', usersRouter) se connect karo
JavaScript — routes/users.js 
// routes/users.js
const express = require('express');
const router = express.Router(); // nayi Router instance

// GET /api/users
router.get('/', (req, res) => {
  res.send('Saare users ki list');
});

// GET /api/users/123
router.get('/:id', (req, res) => {
  res.send(`User ID: ${req.params.id}`);
});

// POST /api/users
router.post('/', (req, res) => {
  res.status(201).send('Naya user ban gaya!');
});

module.exports = router; // export karo
JavaScript — app.js (main file) 
// app.js
const express = require('express');
const app = express();

const usersRouter = require('./routes/users');

app.use(express.json()); // JSON parsing

// /api/users pe saare users routes mount ho jayenge
app.use('/api/users', usersRouter);

app.listen(3000, () => {
  console.log('Server chal raha hai port 3000 pe!');
});
💡
SAMJHO AISE
Jab app.use('/api/users', usersRouter) likhte ho, toh usersRouter ke andar jo / hai, woh actually /api/users ban jaata hai. Aur /:id ban jaata hai /api/users/:id.

App Architecture — Sahi Structure

Professional apps aise organize hoti hain

Jaise ghar mein alag alag kamre hote hain (bedroom, kitchen, bathroom) — Express app mein bhi alag alag folders hote hain har kaam ke liye. Isse Separation of Concerns kehte hain.

Directory Structure 
my-app/
├── app.js          # Main file — sab yahan shuru hota hai
├── package.json
├── routes/         # Sirf routes define karo
│   ├── users.js
│   └── products.js
├── controllers/    # Business logic yahan
│   ├── usersController.js
│   └── productsController.js
├── models/         # Database structure
│   ├── User.js
│   └── Product.js
├── middleware/     # Custom middleware
│   ├── auth.js
│   └── logger.js
├── config/         # Settings aur environment variables
│   └── index.js
└── public/         # Static files (HTML, CSS, images)
    └── index.html
🚪
app.js
Entry point hai. Express setup, database connect, global middleware, aur routers mount karna — sab yahan hota hai.
🗺️
routes/
Sirf URLs define karo. Controller functions ko call karo. Business logic yahan nahi aati. Clean aur simple.
🧠
controllers/
Asli kaam yahan hota hai. Data process karo, model se baat karo, response tayyar karo.
🗃️
models/
Database ka structure yahan define hota hai. Mongoose schemas, queries — sab data-related kaam.
🔧
middleware/
Reusable middleware functions. Auth check, logging, validation — baar baar use hone waale functions.

Request ka Safar — Step by Step

Client se server aur wapas — complete journey

Chalein dekhte hain — jab koi browser GET /api/users/123 request bhejta hai, toh kya hota hai? Har step carefully samjho.

1
🌐 Client ne request bheji: GET /api/users/123
↓
2
📥 app.js ne request receive ki
↓
3
🔄 Global Middleware chalye — express.json(), logging, etc.
↓
4
🗺️ app.js ne path /api/users match kiya → usersRouter pe bhej diya
↓
5
🔒 Router Middleware — authentication check hua
↓
6
🎯 Router ne /:id match kiya → getUserById() controller call hua
↓
7
🗃️ Controller ne User.findById(123) se database se data manga
↓
8
✅ Controller ne res.json(userData) se response bheja
↓
9
🌐 Client ko JSON data mila!
🎓
EXAM TIP — Request Flow
Exam mein agar poochha jaye "Express app mein request kaise travel karta hai?" — yaad rakho: Client → Global Middleware → Router → Router Middleware → Route Handler → Controller → Model → Response. Yeh sequence bahut important hai!
