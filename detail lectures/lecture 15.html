Unit Testing — Kya Hai Yeh?
Code likhne ke baad isko verify karna ke sahi kaam kar raha hai

Socho tumne ek calculator banaya. Ab tum khud se check karte ho — 2+2 = 4 aa raha hai ya nahi, 10/0 pe error aa rahi hai ya nahi. Yeh check karna hi Testing hai. Jab yeh check code ke zariye automatically ho, toh usse Unit Testing kehte hain.

Unit matlab ek choti si cheez — ek function, ek component, ek method. Har unit ko alag alag test karte hain.

🎯

Bugs Pehle Pakdo

Production mein jaane se pehle bugs dhundhte hain. Customer ko kharab experience nahi milta.

♻️

Safe Refactoring

Code change karo — agar test pass hon to confirm hai ke kuch tuta nahi. Bina darr ke improve karo.

📄

Documentation

Tests batate hain ke code kya karta hai. Doosra developer padhke samajh jaata hai.

⚡

Fast Feedback

Seconds mein pata chal jaata hai ke kuch tuta hai. Manual testing mein ghante lagte hain.

TypeKya test karta haiSpeedTool
Unit TestEk function / component⚡ FastestJest + RTL
Integration TestMultiple components saath🚀 FastJest + RTL
End-to-End TestPoori app (browser mein)🐢 SlowCypress, Playwright
📝 Exam Tip

Testing pyramid: Zyada Unit Tests, kam Integration Tests, aur bahut kam E2E Tests. Unit tests fast aur cheap hote hain isliye zyada likhte hain.

Jest — Testing Framework
React ka default testing tool — Facebook ne banaya

Jest ek JavaScript Testing Framework hai. Create React App mein yeh already installed hota hai. Yeh humara test runner hai — yani yeh sare tests chalata hai aur batata hai kaunse pass hue, kaunse fail.

📋 Jest ke main kaam:

  • test() / it() — ek test case define karta hai
  • describe() — tests ko group karta hai (jaise ek folder)
  • expect() — kisi value ke baare mein expectation likhta hai
  • Matchers — toBe, toEqual, toContain, toBeNull wagerah
  • beforeEach / afterEach — har test se pehle/baad setup/cleanup
📄 math.js + math.test.jsJest
// math.js — yeh function test hoga
export function add(a, b) { return a + b; }
export function multiply(a, b) { return a * b; }
export function divide(a, b) {
  if (b === 0) throw new Error('Zero se divide nahi kar sakte!');
  return a / b;
}

// math.test.js — test file (.test.js ya .spec.js)
import { add, multiply, divide } from './math';

describe('Math Functions', () => {

  test('2 aur 3 ka sum 5 hona chahiye', () => {
    const result = add(2, 3);
    expect(result).toBe(5); // exact match
  });

  test('4 aur 5 ka product 20 hoga', () => {
    expect(multiply(4, 5)).toBe(20);
  });

  test('zero se divide pe error aaye', () => {
    expect(() => divide(10, 0)).toThrow('Zero se divide nahi kar sakte!');
  });
});

// Run karo: npm test

React Testing Library (RTL)
React components ko user ki tarah test karo

Jest sirf JavaScript test karta hai. Lekin React components mein HTML render hoti hai — buttons, text, inputs hote hain. React Testing Library (RTL) in components ko browser ki tarah render karke test karta hai. Yeh sochta hai: "User kya dekhta hai aur kya karta hai" — implementation details nahi.

🔑 RTL ke 3 golden rules:

  • render() — component ko virtual DOM mein render karo
  • Query functions — elements dhundho (getByText, getByRole, etc.)
  • userEvent / fireEvent — user actions simulate karo (click, type, etc.)
📄 Counter.js + Counter.test.jsRTL + Jest
// Counter.js — test hone wala component
import React, { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);
  return (
    <div>
      <h2>Count: {count}</h2>
      <button onClick={() => setCount(count + 1)}>Barha</button>
      <button onClick={() => setCount(count - 1)}>Ghata</button>
    </div>
  );
}
export default Counter;

// Counter.test.js
import { render, screen, fireEvent } from '@testing-library/react';
import Counter from './Counter';

describe('Counter Component', () => {

  test('shuruaat mein 0 dikhna chahiye', () => {
    render(<Counter />);
    expect(screen.getByText('Count: 0')).toBeInTheDocument();
  });

  test('Barha button click karne par count +1 ho', () => {
    render(<Counter />);
    fireEvent.click(screen.getByText('Barha'));
    expect(screen.getByText('Count: 1')).toBeInTheDocument();
  });

  test('Ghata click par count -1 ho', () => {
    render(<Counter />);
    fireEvent.click(screen.getByText('Ghata'));
    expect(screen.getByText('Count: -1')).toBeInTheDocument();
  });
});

Query Functions — Elements Dhundhne ke Tariqe

💡 Best Practice

getByRole() use karna sab se acha hai — yeh accessibility ke liye bhi check karta hai. getByTestId() sirf last resort par use karo.

// Async API test — data aane ka intezaar karo
test('API se students load hon', async () => {
  render(<StudentList />);

  // Loading dikhna chahiye pehle
  expect(screen.getByText('Loading...')).toBeInTheDocument();

  // Students aane ka wait karo

  expect(student).toBeInTheDocument();

// Form submit test
test('login form sahi kaam kare', async () => {
  render(<LoginForm />);

  // Input fields mein type karo
  fireEvent.change(screen.getByLabelText('Email'), {
    target: { value: 'ali@test.com' }
  });
  fireEvent.change(screen.getByLabelText('Password'), {
    target: { value: 'secret123' }
  });

  // Submit karo
  fireEvent.click(screen.getByRole('button', { name: 'Login' }));

  // Success message expect karo
  await screen.findByText('Login successful!');
});

// Snapshot test
test('Header UI match kare snapshot se', () => {
  const { asFragment } = render(<Header title="Home" />);
  expect(asFragment()).toMatchSnapshot();

Mocking — Duplicate Banana
Real API calls ki jagah fake responses use karna tests mein

Socho tum ek restaurant ka test kar rahe ho. Real chef ko bulana expensive hai — isliye ek fake chef (mock) banate ho jo same kaam kare. Testing mein bhi real API call karne ki zaroorat nahi — fake response de do. Iski wajah: API slow hai, cost money karti hai, ya abhi bani hi nahi.

📄 Mocking ExamplesJest Mock
// 1. Function mock karna
const mockFn = jest.fn();
mockFn('Ali');
expect(mockFn).toHaveBeenCalledWith('Ali');
expect(mockFn).toHaveBeenCalledTimes(1);

// 2. Module mock — axios ko mock karo
jest.mock('axios');
import axios from 'axios';

test('API se students fetch ho', async () => {
  // Fake response set karo
  axios.get.mockResolvedValue({
    data: [{ id: 1, name: 'Ali' }, { id: 2, name: 'Sara' }]
  });

  render(<StudentList />);

  expect(await screen.findByText('Ali')).toBeInTheDocument();
  expect(screen.getByText('Sara')).toBeInTheDocument();
  expect(axios.get).toHaveBeenCalledWith('/api/students');
});

// 3. Error case mock karna
test('API error pe error message dikhaye', async () => {
  axios.get.mockRejectedValue(new Error('Network Error'));
  render(<StudentList />);
  const err = await screen.findByText(/kuch galat hua/i);
  expect(err).toBeInTheDocument();
});
⚠️ Dhyan Do

Mock ke baad cleanup zaroori hai. afterEach(() => jest.clearAllMocks()) ya jest.resetAllMocks() use karo warna ek test ka mock dusre test ko affect karega.

MongoDB — Data ka Ghar
JSON jaisi documents mein data store karna

MongoDB ek NoSQL database hai. Normal SQL databases mein tables aur rows hote hain — MongoDB mein Collections aur Documents hain. Document bilkul JavaScript object jaisa lagta hai (JSON format).

📋 MongoDB vs SQL:

  • SQL: Table = MongoDB: Collection
  • SQL: Row = MongoDB: Document
  • SQL: Column = MongoDB: Field
  • SQL: JOIN = MongoDB: $lookup / Populate

Express.js — REST API Banana
Frontend ke liye API endpoints banana

Express ek web framework for Node.js hai. Yeh HTTP requests (GET, POST, PUT, DELETE) handle karta hai. React jab data maangta hai, Express usse serve karta hai — jaise waiter restaurant mein order leta hai aur khana deta hai.

📄 server.js + routes/students.jsExpress.js
// server.js — main entry point
const express = require('express');
const cors = require('cors');
require('./db'); // MongoDB connect

const app = express();
app.use(cors());           // React se request aane dena
app.use(express.json()); // JSON body parse karna

app.use('/api/students', require('./routes/students'));

app.listen(5000, () => console.log('Server port 5000 par chal raha hai'));

// routes/students.js — CRUD routes
const router = require('express').Router();
const Student = require('../models/Student');

// GET — sare students lao
router.get('/', async (req, res) => {
  try {
    const students = await Student.find();
    res.json(students);
  } catch (err) { res.status(500).json({ message: err.message }); }
});

// POST — naya student add karo
router.post('/', async (req, res) => {
  const student = new Student(req.body);
  try {
    const saved = await student.save();
    res.status(201).json(saved);
  } catch (err) { res.status(400).json({ message: err.message }); }
});

// PUT — student update karo
router.put('/:id', async (req, res) => {
  const updated = await Student.findByIdAndUpdate(req.params.id, req.body, { new: true });
  res.json(updated);
});

// DELETE — student hatao
router.delete('/:id', async (req, res) => {
  await Student.findByIdAndDelete(req.params.id);
  res.json({ message: 'Student delete ho gaya' });
});

module.exports = router;

React — Frontend ka Kaam
Backend API se connect karke UI banana

React ka frontend Express ke API se fetch ya axios se baat karta hai. User kuch kare — form fill kare, button dabaaye — React API call kare, data aaye, UI update ho.

📄 StudentList.jsx — Full ComponentReact
import React, { useState, useEffect } from 'react';
import axios from 'axios';

const API = 'http://localhost:5000/api/students';

function StudentList() {
  const [students, setStudents] = useState([]);
  const [loading, setLoading]   = useState(true);
  const [name, setName]         = useState('');
  const [email, setEmail]       = useState('');

  // Mount hone par students fetch karo
  useEffect(() => { fetchStudents(); }, []);

  const fetchStudents = async () => {
    const res = await axios.get(API);
    setStudents(res.data);
    setLoading(false);
  };

  const addStudent = async (e) => {
    e.preventDefault();
    await axios.post(API, { name, email, roll: Date.now() });
    setName(''); setEmail('');
    fetchStudents(); // Refresh list
  };

  const deleteStudent = async (id) => {
    await axios.delete(`${API}/${id}`);
    fetchStudents();
  };

  if (loading) return <p>Loading...</p>;

  return (
    <div>
      <form onSubmit={addStudent}>
        <input value={name} onChange={e => setName(e.target.value)} placeholder="Naam" />
        <input value={email} onChange={e => setEmail(e.target.value)} placeholder="Email" />
        <button type="submit">Add Student</button>
      </form>
      {students.map(s => (
        <div key={s._id}>
          <span>{s.name} — {s.email}</span>
          <button onClick={() => deleteStudent(s._id)}>Delete</button>
        </div>
      ))}
    </div>
  );
}
export default StudentList;
