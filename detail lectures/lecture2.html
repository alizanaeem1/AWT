01

Node.js Kya Hai?

Pehle samjhein ke Node.js actually hai kya — kyunki bahut se students sochte hain ye ek programming language hai. Nahi! Ye ek runtime environment hai.

🌐
Open-Source Hai
Node.js bilkul free hai. Koi bhi use kar sakta hai, modify kar sakta hai. Duniya bhar ke developers isko improve karte rehte hain.
FREE TO USE
⚙️
Runtime Environment
JavaScript sirf browser mein chalti thi — lekin Node.js ki wajah se aap JavaScript ko server par bhi chala sakte hain. Browser ke bahar!
SERVER-SIDE JS
🏎️
V8 Engine
Google Chrome ka V8 engine use karta hai. Ye engine JavaScript code ko seedha machine code mein convert karta hai — isiliye bahut fast hota hai.
GOOGLE V8
💻
Cross-Platform
Windows pe bhi, Linux pe bhi, Mac pe bhi — Node.js har jagah chalta hai. Ek baar likho, kahin bhi chalao.
WINDOWS · LINUX · MAC
💡
Simple samajhne wali baat: Sochein ke pehle JavaScript sirf ghar (browser) ke andar kaam karti thi. Node.js ne JavaScript ko ghar se bahar — yani server tak — pohoncha diya!
Terminal — Node.js Version Check 
# Node.js install hai ya nahi check karein:
node --version    # Output: v20.x.x
npm  --version    # Output: 10.x.x

# Pehla Node.js program chalayein:
node hello.js
🎓
Exam Tip: "Node.js ek programming language hai" — ye GALAT hai. Node.js ek JavaScript Runtime Environment hai jo V8 engine pe based hai. Exam mein ye zaroor aata hai!
02

Core Features

Ab samjhein ke Node.js ItNa popular kyun hai — in 4 main features ki wajah se Node.js bahut powerful aur fast hai.

🔀
1. Asynchronous & Non-Blocking — "Ruka Nahi Karta"

Ye Node.js ka sabse bada feature hai. Normal programming mein ek kaam hoga tab doosra shuru hoga — Node.js yeh nahi karta. Ye ek saath multiple kaam karta rehta hai.

❌ Blocking (Puraani Style)

Waiter order leta hai → kitchen mein khana banne tak wahan khara rehta hai → khana lata hai → phir doosre customer ke paas jata hai.

Result: Customers wait karte hain!

✅ Non-Blocking (Node.js Style)

Waiter order leta hai → doosre customer ke paas chala jata hai → jab khana ready hota hai tab serve karta hai.

Result: Sabhi customers khush!

JavaScript — Async Example 
// Asynchronous code ka example
console.log("1 — Pehla kaam shuru");

setTimeout(function() {
  console.log("2 — File read hui (2 second baad)");
}, 2000);

console.log("3 — Teesra kaam (2 ka wait nahi kiya!)");

// Output Order:
// 1 — Pehla kaam shuru
// 3 — Teesra kaam (2 ka wait nahi kiya!)
// 2 — File read hui (2 second baad)
🧵
Single-Threaded
Node.js ek hi thread use karta hai — lekin event-driven architecture ki wajah se hazaron requests handle kar sakta hai. Ek engine — hazaron kaam.
🚀
Fast Performance
V8 engine JavaScript ko seedha machine code mein convert karta hai — interpreter ka chakkar nahi. Isiliye bahut tez hota hai.
📡
Real-time Apps
Chat apps, live notifications, online games — jo cheezein real-time data chahti hain, Node.js unke liye best choice hai.
⚠️
Dhyan rakhein: Node.js CPU-intensive kaam (video processing, badi calculations) ke liye theek nahi. Ye I/O operations (file read/write, database, network) ke liye best hai.
03

Node.js Architecture

Architecture matlab — Node.js andar se kaam kaise karta hai? Client se request aayi, kahan gayi, kya hua, response kab aaya? Ye poora cycle samjhein.

🏗️
Node.js ka Model: Node.js teen cheezein follow karta hai — Event-Driven (events pe kaam karta hai), Non-Blocking I/O (ruka nahi karta), aur Client-Server Architecture (client request karta hai, server jawaab deta hai).

Request ka Safar — Step by Step:

1
Client Request Bhejta Hai

Browser ya mobile app server ko request bhejta hai — jaise "mujhe yeh page do"

2
Event Queue Mein Jata Hai

Request seedha process nahi hoti — pehle event queue mein line mein lagti hai

3
Event Loop Check Karta Hai

Event loop har waqt check karta rehta hai — koi kaam pending hai? Agar ha, process karo

4
Heavy Kaam Background Thread Ko

File padhna ya database query — ye kaam background mein hota hai taake main thread ruka na rahe

5
Response Client Ko Bheja

Kaam complete hone par response wapas client ko bhej diya jata hai

JavaScript — Simple HTTP Server 
const http = require('http');  // http module load karo

// Server banao
const server = http.createServer((req, res) => {
  res.writeHead(200, { 'Content-Type': 'text/plain' });
  res.end('Assalam o Alaikum! Ye mera server hai.');
});

// Port 3000 pe sunna shuru karo
server.listen(3000, () => {
  console.log('Server chalu hai: http://localhost:3000');
});
✅
Real Life Example: Architecture ko restaurant ki tarah samjhein — Customer (Client) order deta hai, order pad (Event Queue) pe likha jata hai, manager (Event Loop) order check karta hai, kitchen (Background Thread) mein khaana banta hai, phir customer ko serve hota hai.
04

Node.js Modules

Module matlab — code ka ek alag hissa. Jaise ek kitaab mein chapters hote hain, waise hi Node.js mein modules hote hain. Har module ek specific kaam karta hai.

📦
Yaad rakhein: Node.js mein har file ek module hoti hai. Aap require() se module load karte hain aur module.exports se bahar nikaalte hain.
🔧
Core Modules
Node.js ke sath already aate hain — install karne ki zaroorat nahi.

Examples:
• fs — File padhna/likhna
• http — Web server banana
• path — File paths handle karna
• os — Operating system info
BUILT-IN
✍️
Local Modules
Aap khud banate hain — apna code organize karne ke liye.

Faida: Code ek jagah hone ke bajaye alag-alag files mein hota hai — samajhna aur manage karna asaan.
KHUD BANAO
📥
Third-Party Modules
NPM se download karke use karte hain.

Examples:
• express — Web framework
• mongoose — MongoDB ke liye
• nodemon — Auto restart
NPM INSTALL
JavaScript — Modules Ka Use 
// ===== 1. Core Module Use Karna =====
const fs = require('fs');  // Install nahi, bas require karo

fs.readFile('data.txt', 'utf8', (err, data) => {
  console.log(data);  // File ka content print karo
});

// ===== 2. Local Module Banana =====
// math.js file mein likho:
function jama(a, b) { return a + b; }
module.exports = { jama };

// app.js mein use karo:
const math = require('./math');  // ./ matlab same folder
console.log(math.jama(5, 3));  // Output: 8
Module Type Install Zaroori? Kahan Se Aata Hai? Example
Core Module Nahi Node.js ke saath built-in fs, http, path
Local Module Nahi Aap khud banate hain ./myModule
Third-Party Haan npm install se express, mongoose
🎓
Exam Tip: Core modules install nahi karne padte — ye Node.js ke andar already hote hain. Jab question aaye "Which module is used to create a server?" — answer hai http.
05

Event Loop, Callbacks & Emitters

Ye Node.js ka "dil" hai. Agar ye samajh gaye toh Node.js ki poori asynchronous power samajh mein aa jaayegi.

🔄
Event Loop — Traffic Police Wala
Event Loop Node.js ka woh hissa hai jo continuously check karta rehta hai:
• Call Stack — Koi kaam abhi chal raha hai?
• Event Queue — Koi kaam pending hai?
• Callback Queue — Koi purana kaam complete hua?

Jaise traffic police officer decide karta hai kaun sa traffic pehle jayega — Event Loop decide karta hai kaunsa request pehle process ho.
📞
Callback — "Kaam Hone Par Bulana"
Callback ek function hai jo doosre function ko diya jata hai — aur jab kaam complete ho tab ye function call hota hai.

Real Life: Aapne restaurant mein order diya. Aap keh dete hain "jab khana ready ho tab mujhe bata dena" — ye callback hai. Aap wahan khare nahi rehte — dusra kaam karte rehte hain.
JavaScript — Callback Example 
const fs = require('fs');

// File padhna + callback function
fs.readFile('data.txt', 'utf8', function(err, result) {
  // Ye function tab chalega jab file read ho jayegi
  if (err) {
    console.log('Error aaya!', err);
  } else {
    console.log('File ka content:', result);
  }
});

console.log('File read ho rahi hai...');
// Ye line pehle chalegi! Callback baad mein.
📢
Event Emitter — "Awaz Lagana aur Sunna"
Event Emitter Observer Pattern pe kaam karta hai. Koi event "emit" (broadcast) karta hai — aur jis ne "listen" kar rakha hai woh react karta hai.

Examples: File upload complete hui → event emit → doosra kaam shuru. Button click hua → event → function chala.
JavaScript — Event Emitter 
const EventEmitter = require('events');
const myEmitter = new EventEmitter();

// Event ko "listen" karo (observer)
myEmitter.on('khaanaReady', (naam) => {
  console.log(`${naam} ka khaana ready hai!`);
});

// Event "emit" karo (trigger karo)
myEmitter.emit('khaanaReady', 'Ali');
// Output: Ali ka khaana ready hai!
Cheez Kya Karta Hai? Real Life Example
Event Loop Queue check karta hai, kaam assign karta hai Traffic Police Officer
Callback Kaam hone ke baad call hone wala function Restaurant mein "Hona pe bulao"
Event Emitter Event broadcast karta hai School ka bell bajana

NPM — Node Package Manager

NPM duniya ki sabse badi software registry hai. Agar koi feature chahiye toh khud banana zaroori nahi — NPM pe already available hai. Bas install karo aur use karo!

📦
NPM Kya Hai?
Node Package Manager — yaani node ke packages manage karne wala tool. Jaise Google Play se apps download karte hain, NPM se Node.js packages download karte hain.
🗂️
package.json
Ye aapke project ki "shopping list" hai — project ka naam, version, aur kaunse packages use ho rahe hain — sab kuch is file mein hota hai.
🌍
NPM Registry
Online database jahan 2 million se zyada packages available hain. Express, mongoose, lodash, socket.io — sab yahan milte hain.
Terminal — NPM Commands 
# Naya project shuru karo (package.json banega)
npm init

# Express install karo
npm install express

# Aik saath kai packages install karo
npm install express mongoose nodemon

# Package remove karo
npm uninstall express

# NPM version dekho
npm -v

# Saare installed packages list karo
npm list

package.json — Samjhein:

JSON — package.json 
{
  "name": "mera-project",         // Project ka naam
  "version": "1.0.0",           // Version number
  "description": "Mera app",   // Mukhtasar taaruf
  "main": "index.js",           // Entry point file
  "scripts": {
    "start": "node index.js",   // npm start se chalega
    "dev": "nodemon index.js"    // Development ke liye
  },
  "dependencies": {
    "express": "^4.18.0"        // Required packages
  }
}
Command Kya Karta Hai?
npm init Naya project shuru karo, package.json banega
npm install Saare dependencies install karo (package.json se)
npm install [naam] Ek specific package install karo
npm uninstall [naam] Package remove karo
npm -v NPM version check karo
npm start Project ko chalaao (package.json script se)
✅
Pro Tip: Hamesha node_modules folder ko .gitignore mein daalo. Ye folder share nahi karte — dusra banda npm install se sab kuch dobara install kar sakta hai.
