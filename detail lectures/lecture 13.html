Part A · 0–10 min

React Kyun Seekhen? 🤔

Pehle yeh samjho ke React hai kya aur hum ise use kyun karte hain.

💬 Simple Explanation

  • React ek JavaScript library hai jo hume UI (User Interface) banana mein help karta hai.
  • React mein pura app chote chote components se banta hai — bilkul Lego blocks ki tarah!
  • Data upar se neeche jaata hai (props), aur events neeche se upar (callbacks).
  • Jab state change hoti hai, React sirf woh hissa update karta hai jo badla — pura page reload nahi hota!
Real World 📱

Instagram Feed

Instagram ka feed dekho — har post ek component hai. Like button click karo toh sirf woh component update hota hai, pura page reload nahi hota. Yahi React ki power hai!

Core Idea 🌳

Component Tree

React app ek tree ki tarah hota hai. App component sab ka parent hai, phir uske andar chhote chhote components hote hain jaise Navbar, Feed, Sidebar wagera.

Key Concept ⚡

Virtual DOM

React pehle ek imaginary DOM banata hai memory mein. Phir compare karta hai real DOM se. Sirf jo different hai, woh update karta hai. Isliye bahut fast hai!

💡
Teacher Tip: Class mein pooch: "Kahan kahan repeated UI patterns dekhte ho?" (cards, product tiles, posts, notifications). Students ko khud React ki zaroorat feel hogi!
Part B · 10–30 min

JSX — React Ki Zubaan ✍️

JSX ek aisi syntax hai jo HTML aur JavaScript ko milati hai. Lekin HTML nahi hai — JS rules follow karta hai!

🎯
Simple Samajhna: JSX bilkul HTML jaisa dikhta hai, lekin yeh asal mein JavaScript hai. Browser isko samajhne ke liye React.createElement() mein badal deta hai.
Rule 1 ☝️

Ek Parent Element

JSX mein hamesha sirf ek parent element hona chahiye. Do sibling elements directly return nahi kar sakte. <>...</> Fragment use karo agar extra div nahi chahiye.

Rule 2 🎨

className, class nahi!

HTML mein class likhte hain CSS ke liye, lekin JSX mein className likhna hai. Yeh isliye ke class JavaScript ka reserved word hai!

Rule 3 { }

JS Expressions — { }

JavaScript ka koi bhi expression curly braces { } mein likho. Jaise variables, functions, ternary operators wagera. Seedha if nahi chal sakta!

Rule 4 🖌️

Inline Style Object

HTML mein string dete hain: style="color:red" lekin JSX mein object dena hota hai: style={{ color: 'red' }} — double curly braces!

📊 HTML vs JSX Farq

Feature HTML JSX
CSS Class class="box" className="box"
Click Event onclick="f()" onClick={f}
Text + Variable "hello " + name {"hello " + name}
Inline Style style="color:red" style={{ color: 'red' }}
List Rendering Manually .map() + key

💻 JSX ka Sahih Tarika

JSX · Greeting.jsx 
// Yeh ek simple JSX component hai
function Greeting() {
  const name = "Ali";
  const isLoggedIn = true;

  return (
    <div className="card">
      {/* JavaScript expression curly braces mein */}
      <h1>Hello, {name}!</h1>
      
      {/* Ternary use karo — seedha if nahi chalta */}
      <p>{isLoggedIn ? "Welcome back!" : "Please login"}</p>
      
      {/* Inline style = object mein */}
      <button 
        style={{ backgroundColor: "navy", color: "white" }}
        onClick={() => alert("Clicked!")}
      >
        Click Karo
      </button>
    </div>
  );
}
JSX · List Rendering (key zaroor do!) 
const fruits = ["Apple", "Mango", "Banana"];

function FruitList() {
  return (
    <ul>
      {fruits.map((fruit, index) => (
        {/* KEY dena ZAROOR hai — React track karta hai */}
        <li key={index}>{fruit}</li>
      ))}
    </ul>
  );
}
⚠️
Common Galtiyan: ❌ Do elements seedha return karna (wrapper ke baghair)
❌ JSX ke andar seedha if use karna (ternary ya && use karo)
❌ .map() mein key bhool jaana
📚
Exam Tip: Agar puchha jaye "JSX kyun use karte hain?" toh sirf "React ko chahiye" mat kaho. Bolо: JSX readability improve karta hai aur component composition asaan banata hai.

Part D · 50–70 min

Components, Props & State 🧩

Components reusable UI blocks hain. Props = input. State = component ka apna data.

Props 📨

Props — Parent se Data

Props parent component se child mein pass hota hai. Props read-only hote hain — child inhe change nahi kar sakta. Bilkul function ke arguments ki tarah.

State 🔄

State — Component Ka Data

State component ka apna data hai jo change ho sakta hai. Jab state change hoti hai, React us component ko dobara render karta hai — UI update ho jaata hai!

JSX · Props + State Example 
import { useState } from 'react';

// Product Card — Props se data leta hai
function ProductCard({ name, price }) {
  // State — cart mein add hua ya nahi
  const [inCart, setInCart] = useState(false);

  return (
    <div className="product-card">
      <h3>{name}</h3>
      <p>Rs. {price}</p>
      
      <button onClick={() => setInCart(!inCart)}>
        {inCart ? "✅ Cart Mein Hai" : "🛒 Cart Mein Dalo"}
      </button>
    </div>
  );
}

// Parent component props pass karta hai
function App() {
  return (
    <ProductCard name="Laptop" price={85000} />
  );
}
⚠️
Common Galtiyan: ❌ State ko seedha mutate karna — arr.push(item) mat karo! Hamesha new array banao: setArr([...arr, item])
❌ Props pass karna bhool jaana, phir undefined use karna
📚
Exam Tip — Classic Question: "Props aur State mein farq kya hai?" — Props parent se aata hai aur read-only hai. State component ke andar hoti hai aur change ho sakti hai. Yeh do lines yaad kar lo!
Part E · 70–85 min

React Router — Pages Banana 🗺️

React Router se hum SPA (Single Page Application) mein multiple pages ka effect dete hain bina page reload ke!

🌐
Simple Samajhna: Normal website mein link click karo toh pura page reload hota hai — server se naya HTML aata hai. React Router mein sirf component change hota hai — page reload bilkul nahi!
01

Install Karo

npm install react-router-dom — yeh package install karo.

02

BrowserRouter Wrap Karo

main.jsx mein puri App ko <BrowserRouter> mein wrap karo — yahi routing enable karta hai.

03

Routes Define Karo

App.jsx mein <Routes> aur <Route path="/" element={<Home/>} /> likho.

04

Link Se Navigate Karo

HTML ke <a> tag ki jagah <Link to="/about"> use karo — yeh reload nahi karta!

JSX · React Router Setup — App.jsx 
import { Routes, Route, Link } from 'react-router-dom';
import Home from './pages/Home';
import About from './pages/About';
import NotFound from './pages/NotFound';

function App() {
  return (
    <>
      {/* Navigation links — page reload nahi karte */}
      <nav>
        <Link to="/">Home</Link>
        <Link to="/about">About</Link>
      </nav>

      {/* Yahan matching component render hoga */}
      <Routes>
        <Route path="/"        element={<Home />} />
        <Route path="/about"    element={<About />} />
        <Route path="*"         element={<NotFound />} />
        {/* path="*" = koi bhi aur route match na ho toh */}
      </Routes>
    </>
  );
}
📚
Exam Tip: "No page reload" + "fast navigation" = Client-side routing (React Router). Yeh answer yaad rakh!
Part F · 85–105 min

Forms — Controlled vs Uncontrolled 📝

React mein forms do tareekon se handle hote hain. Dono ka comparison yahan karo.

Controlled Form 🎛️

React State se Control

React state hi data ka source hai. Har keystroke pe onChange fire hota hai aur state update hoti hai. React hamesha jaanta hai input mein kya hai.


✅ Pros: Easy validation, instant UI updates, predictable

❌ Cons: Bahut saare fields ho toh zyada code

Uncontrolled Form 🔗

DOM se Direct Read

Input DOM mein khud store hota hai. Submit pe useRef() se value read karte hain. React ko pata nahi hota input mein kya chal raha hai.


✅ Pros: Less re-renders, simple forms ke liye quick

❌ Cons: Validation mushkil, kam React-ish

FeatureControlledUncontrolled
Data SourceReact StateDOM
ValidationAsaanManual (mushkil)
Re-render per keystrokeHaan ✅Nahi ❌
Best ForComplex formsSimple/quick forms
Value Accessstate valueref.current.value
JSX · Controlled Form — Login 
function LoginForm() {
  const [email, setEmail]       = useState("");
  const [password, setPassword] = useState("");
  const [error, setError]       = useState("");

  function handleSubmit(e) {
    e.preventDefault(); // page reload rokna
    if (!email || !password) {
      setError("Email aur password dono zaroori hain!");
      return;
    }
    setError("");
    console.log("Login:", email, password);
  }

  return (
    <form onSubmit={handleSubmit}>
      <input 
        value={email}
        onChange={e => setEmail(e.target.value)} 
        placeholder="Email"
      />
      {error && <p style={{ color: 'red' }}>{error}</p>}
      <button>Login</button>
    </form>
  );
}
💡
Default Choice: Hamesha Controlled form use karo. Uncontrolled sirf special cases mein use karo — jaise kisi legacy system ke saath ya bahut simple single input ke liye.
Part G · 105–115 min

Hooks — React Ki Superpowers 🪝

Hooks functional components mein state, side effects, refs aur context add karte hain.

useState
Component ka memory. Jab value change ho aur UI update chahiye toh useState use karo.
🔸 Counter, toggle, form input
useEffect
Side effects ke liye — API call karna, subscriptions, localStorage — render ke baad chalta hai.
🔸 Data fetch, timer, event listeners
useRef
DOM element access karo ya koi value yaad rakho bina re-render kiye. Input focus karna.
🔸 Input focus, interval ID store
useContext
Global data share karo bina props drilling ke. Theme, user info wagera poori app mein.
🔸 Dark/light theme, user auth
JSX · useEffect + useRef Example 
import { useState, useEffect, useRef } from 'react';

function Notes() {
  const [notes, setNotes] = useState([]);
  const [text, setText]   = useState("");
  const inputRef          = useRef(null);

  // useEffect — sirf pehli baar chalta hai (mount pe)
  useEffect(() => {
    const saved = localStorage.getItem("notes");
    if (saved) setNotes(JSON.parse(saved));
  }, []); // [] = sirf ek baar chalao

  // notes change hone pe localStorage update karo
  useEffect(() => {
    localStorage.setItem("notes", JSON.stringify(notes));
  }, [notes]); // [notes] = jab notes badle tab chalao

  function addNote() {
    if (!text) return;
    setNotes([...notes, text]);
    setText("");
    inputRef.current.focus(); // useRef — input pe focus
  }

  return (
    <div>
      <input ref={inputRef} value={text} 
             onChange={e => setText(e.target.value)} />
      <button onClick={addNote}>Add Note</button>
    </div>
  );
}
⚡
Important Note — useEffect: useEffect(() => {'{...}'}, []) — empty array = sirf ek baar mount pe chale
useEffect(() => {'{...}'}, [value]) — value change hone pe chale
useEffect(() => {'{...}'}) — har render ke baad chale (usually avoid karo!)
