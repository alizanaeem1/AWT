MEAN Stack Kya Hota Hai?
Part A · 0–10 min

MEAN ek full-stack technology combination hai jisme 4 cheezein hoti hain. Socho jaise ek factory ka pipeline — kaam ek jagah se shuru hota hai aur doosri jagah khatam hota hai. React UI pe user kuch karta hai → Express/Node process karta hai → MongoDB mein data save hota hai.

⚛️
M — MongoDB
Database hai. Jahan saara data save hota hai. JSON jaisi format mein data rakhta hai. Jaise ek bada digital almirah.
🟢
E — Express.js
Backend web framework hai. Node ke upar kaam karta hai. Routes banata hai — matlab kaunsa URL request pe kya karna hai.
🟡
A — Angular / React
Frontend UI framework. Is lecture mein hum React use kar rahe hain. Jo user screen pe dekhta hai, woh React banata hai.
⬡
N — Node.js
JavaScript ko server pe chalata hai. Browser ke bahar JS run hoti hai. Express issi ke upar bana hai.

Ab dekho poora request flow kaise kaam karta hai:

User action
→
HTTP Request
fetch / axios
→

→
Mongoose

MongoDB

💡
Important Rule React (frontend) kabhi directly MongoDB se baat nahi karta. Hamesha REST API (Express) ke zariye kaam hota hai. Yeh separation bahut zaroori hai!

REST API Design
Part B · 10–30 min

REST ek convention hai — ek agreement — ke hum API kaise design karein. Socho jaise ek restaurant menu. Har cheez ka alag naam aur order karne ka tarika hota hai. REST mein har "resource" (jaise tasks) ka alag URL hota hai, aur action HTTP method se pata chalta hai.

ℹ️
Naming Rule URL mein verb mat likho! /api/getTasks galat hai. Sahi hai: /api/tasks — method (GET/POST) hi action batata hai.
HTTP Method URL / Route Kya Karta Hai Status Code
GET /api/tasks Saari tasks ki list lo 200 OK
POST /api/tasks Nayi task banao 201 Created
PUT/PATCH /api/tasks/:id Existing task update karo 200 OK
DELETE /api/tasks/:id Task delete karo 200 OK

Real example: Task banane ke liye POST request ka JSON body kuch aisa hoga:

JSON — Request & Response
// POST /api/tasks — Request Body (jo hum bhejte hain) { "title": "Milk khareedna", "done": false } // Server ka Response (jo wapas milta hai) { "_id": "64f3a1b2c9d4e5f6a7b8c9d0", "title": "Milk khareedna", "done": false, "createdAt": "2024-01-15T10:30:00Z" }
⚠️
Common Mistake Agar frontend id expect kare aur backend _id bheje — to app "broken" lagegi lekin dono sides sahi kaam kar rahi hongi! JSON keys same rakho.
🎯
Exam Tip REST mapping table yaad karo: Method + Route + Purpose + Status Code. Yeh theory mein zaroor aata hai!

Express + MongoDB Backend
Part C · 30–55 min

Express routes handle karta hai — matlab kaunse URL pe kya karna hai. Mongoose MongoDB ke saath kaam karta hai — JavaScript objects ko database documents mein convert karta hai. Dono mil kar backend banate hain.

Backend banana ka step-by-step tarika:

  • 1
    npm init -y — Nayi Node project banao. package.json file banega.
  • 2
    Dependencies install karo: npm install express mongoose cors — ye teeno packages chahiye.
  • 3
    server.js banao — Main entry point. Express app yahan setup hoti hai.
  • 4
    MongoDB connect karo — Mongoose.connect() se database se connection banao.
  • 5
    Routes implement karo — GET, POST, PATCH, DELETE endpoints banao.
server.js — Express Backend
const express = require('express'); const mongoose = require('mongoose'); const cors = require('cors'); const app = express(); // ⭐ Middleware — Bina is ke req.body undefined hoga! app.use(express.json()); // ⭐ CORS — React dev server ko allow karo app.use(cors()); // MongoDB se connect karo mongoose.connect('mongodb://localhost/taskdb'); // Task Schema banana const TaskSchema = new mongoose.Schema({ title: { type: String, required: true }, done: { type: Boolean, default: false } }, { timestamps: true }); const Task = mongoose.model('Task', TaskSchema); // GET — Saari tasks lo app.get('/api/tasks', async (req, res) => { const tasks = await Task.find(); res.json(tasks); }); // POST — Nayi task banao app.post('/api/tasks', async (req, res) => { const task = new Task(req.body); await task.save(); res.status(201).json(task); }); app.listen(5000, () => console.log('Server chal raha hai!'));
⚠️
Common Mistake #1 app.use(express.json()) bhool gaye? To req.body undefined aayega aur kuch nahi samjhega kyun nahi chal raha!
⚠️
Common Mistake #2 CORS enable nahi kiya? Browser React se API call block kar dega. Error milega: "CORS policy blocked."

React: Components, Props & State
Part D · 65–85 min

React mein poori UI choti choti components se banti hai — jaise LEGO blocks. Har component apna kaam karta hai. Data do tareeqon se move hoti hai: Props parent se child ko data deti hai, aur State component ka apna internal data hota hai jo change hone par screen update karta hai.

🧩
Component
Ek function jo JSX return kare. TaskList, TaskForm, TaskItem — sab components hain. Chhote pieces milke bari UI banate hain.
📩
Props
Parent se child ko diya gaya data. Read-only hoti hai — child change nahi kar sakta. Jaise function arguments.
🔄
State
Component ka apna data. useState se banate hain. Jab change ho, React screen automatically update karta hai.
Feature Props State
Malik Kaun? Parent component Component khud
Change Ho Sakta? Read-only Setter se change hota
Re-render kab? Jab parent change kare Jab setter call ho
Use kab karein? Data pass karna ho Dynamic data store karna ho
React JSX — TaskApp Example
// TaskApp — Main parent component function TaskApp() { // State — tasks ki list (dynamic data) const [tasks, setTasks] = useState([]); const handleAdd = (newTask) => { setTasks([...tasks, newTask]); // state update = re-render }; return ( <div> {/* Props: onAdd function pass ho raha hai */} <TaskForm onAdd={handleAdd} /> {/* Props: tasks array pass ho raha hai */} <TaskList tasks={tasks} /> </div> ); } // TaskList — Child component, props receive karta hai function TaskList({ tasks }) { return ( <ul> {tasks.map(task => ( {/* key prop zaroor do list mein! */} <TaskItem key={task._id} task={task} /> ))} </ul> ); }
🎯
Exam Tip Classic question: "Props aur State mein farq batao example ke saath." Yaad rakho — Props parent deta hai, State khud ka hota hai!

Hooks & Lifecycle (useEffect)
Part E · 85–105 min

Pehle React mein class components hote the aur lifecycle methods hoti thi. Ab Hooks ne unko replace kar diya. useState se state banate hain, aur useEffect se side effects handle karte hain — jaise API call karna, timer set karna, etc.

🟣
useEffect(() => {}, [])
Empty array [] dene se sirf ek baar chalta hai — jab component pehli baar screen pe aaye. Bilkul componentDidMount jaisa!
🔵
useEffect(() => {}, [value])
Jab value change ho tabhi chalega. Agar list filter change ho to naya data fetch karo.
🟠
useEffect(() => {})
Koi array nahi — Har render pe chalega. Zyadatar infinite loop ka zariya banta hai! Bachke rehna.

API Call ka sahi pattern:

React Hooks — API Fetching
import { useState, useEffect } from 'react'; function TaskApp() { const [tasks, setTasks] = useState([]); const [loading, setLoading] = useState(true); const [error, setError] = useState(null); // Step 1: Page load hote hi tasks fetch karo useEffect(() => { const fetchTasks = async () => { try { setLoading(true); // Loading shuru const res = await fetch('/api/tasks'); const data = await res.json(); // ⭐ await zaroor! setTasks(data); // State update } catch (err) { setError('Tasks load nahi ho sakin!'); } finally { setLoading(false); // Loading khatam } }; fetchTasks(); }, []); // ← Empty array = sirf ek baar if (loading) return <p>Loading...</p>; if (error) return <p>{error}</p>; return <TaskList tasks={tasks} />; }
⚠️
Infinite Loop Bug Agar useEffect ke andar state update karo aur dependency array mein woh state dalo — to infinite loop ho jaata hai! Effect chalta hai → state update → effect phir chalta hai → yeh cycle khatam nahi hoti.

Virtual DOM & Rendering
Part F · 105–115 min

Real DOM (HTML page ka structure) ko directly baar baar update karna bahut slow hota hai. React ne solution nikala: pehle ek Virtual DOM banao — ek lightweight copy memory mein. Jab state change ho, pehle Virtual DOM update karo, phir diff karo (pehle aur abhi mein kya farq hai), aur sirf woh changes Real DOM mein apply karo.

State Change
setTasks(...)
→
New V-DOM
Memory mein
→
Diffing
Kya badla?
→
Patch Real DOM
Sirf farq update
✅
Re-render ≠ Page Reload
Sirf component re-render hota hai. Poora page reload nahi hota. Isliye React bahut fast feel hota hai.
🔑
Key Prop Kyun Zaroori?
List mein key dene se React identify karta hai kaun sa item add/remove hua. Key ke bina React galat element update kar sakta hai!
🧊
State Immutability
State seedha mutate mat karo. Hamesha setter use karo: setTasks([...tasks, newTask]). Spread operator se nayi copy banao.
🎯
Exam Tip "Virtual DOM aur Reconciliation explain karo example ke saath" — yeh descriptive question bahut common hai. Flow yaad rakho: State → V-DOM → Diff → Real DOM patch.
