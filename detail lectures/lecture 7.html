Lecture 1 — Basics

API Aur REST Kya Hai?

Sab se pehle samjhte hain ke API aur REST hote kya hain — real life examples ke saath!

🍕

API Kya Hai?

API (Application Programming Interface) do software systems ke darmiyan communication ka zariya hai. Jaise Foodpanda app server se poochhta hai: "Restaurants dikhao!" — yeh kaam API karta hai.

🌐

REST Kya Hai?

REST (Representational State Transfer) ek style hai web APIs banane ka. Yeh HTTP protocol use karta hai. Simple, fast, aur widely used hai — almost har website ka backend REST API use karta hai.

🏪

Real Life Example

Socho ek restaurant mein: Aap = Client (frontend). Waiter = API. Kitchen = Server (backend). Aap waiter ko order dete ho, waiter kitchen ko batata hai, aur khana wapas lata hai!

💡
Tip — Yaad Rakhein
Jab bhi mobile app koi data dikhaata hai (jaise Facebook posts, YouTube videos), peeche REST API kaam kar raha hota hai!
REST Principles

REST ke 6 Usool

REST ki bunyaad 6 principles par hai. Inhe samajhna bahut zaroori hai!

🔌

1. Client-Server

Frontend aur backend alag alag hote hain. Mobile app (client) alag, server alag. Dono apna kaam khud karte hain.

🏧

2. Stateless

Server kisi ki "yaad" nahi rakhta. Har request mein sab information hona chahiye. Jaise ATM — har baar card daalna parta hai!

💾

3. Cacheable

Response ko cache mein store kar sakte hain. Performance behtar hoti hai. Server par load kam hota hai.

📐

4. Uniform Interface

Standard URLs aur HTTP methods use honge. Consistent design — sab same tareeqe se kaam karega.

🧅

5. Layered System

Alag alag layers ho sakti hain — security layer, database layer, etc. Client ko nahi pata server ke andar kitni layers hain.

📦

6. Code on Demand (Optional)

Server client ko executable code bhi bhej sakta hai (jaise JavaScript). Yeh optional hai.

📝
Exam Tip — Important!
"Stateless" ka matlab hai: Server previous request ko yaad nahi rakhta. Har request independent hoti hai. Yeh concept bahut zyada exam mein aata hai!
HTTP Methods

HTTP Methods — CRUD ka Dil

REST API mein yeh 4 methods sab kuch karte hain — Create, Read, Update, Delete!

Method Kaam Kya Karta Hai? Real Life Example SQL Equivalent
GET Data retrieve karta hai (sirf dekhna) Product list dekhna, profile dekhna SELECT
POST Naya data create karta hai User register karna, order dena INSERT
PUT Pura record update karta hai Profile update karna, address badalna UPDATE
DELETE Record delete karta hai Account delete, post hatana DELETE
⚠️
Common Galti — Login mein GET ya POST?
Login mein POST use hota hai — kyunki password request body mein securely bheja jaata hai. GET use karo toh password URL mein nazar aayega — bilkul unsafe!
Case Study

Endpoints Design — Online Bookstore

Ek Online Bookstore ke liye REST API design karte hain — step by step!

1

Resources Identify Karo

REST mein sab kuch "Resource" hota hai. Bookstore mein 3 resources hain: Users, Books, Orders. Har resource ka apna endpoint hoga.

2

Nouns Use Karo, Verbs Nahi!

❌ /getBooks — yeh galat hai!
✅ /books — yeh sahi hai! HTTP method batata hai kya karna hai, URL sirf resource ka naam hona chahiye.

3

Plural Names Use Karo

❌ /book — singular, galat!
✅ /books — plural, sahi! Standard practice yahi hai.

CRUD Operations

Char CRUD Operations — Poora Code

GET, POST, PUT, DELETE — charon ka complete implementation dekhte hain!

GET — Saare Students 
// GET /api/students — sab students dikhao
app.get('/api/students', (req, res) => {
  res.json(students);  // 200 OK — data bhejo
});
GET — Ek Student by ID 
// GET /api/students/1 — ID se student dhundho
app.get('/api/students/:id', (req, res) => {
  const student = students.find(s => s.id == req.params.id);
  
  if (!student) {
    // Student nahi mila toh 404 bhejo
    return res.status(404).json({ message: "Student nahi mila!" });
  }
  
  res.json(student);  // 200 OK
});
POST — Naya Student Add Karo 
// POST /api/students — naya student banao
app.post('/api/students', (req, res) => {
  const newStudent = {
    id: students.length + 1,  // Auto ID
    ...req.body              // Request se data
  };
  
  students.push(newStudent);
  res.status(201).json(newStudent);  // 201 Created!
});
DELETE — Student Hatao 
// DELETE /api/students/1 — student hatao
app.delete('/api/students/:id', (req, res) => {
  students = students.filter(s => s.id != req.params.id);
  res.json({ message: "Student delete ho gaya!" });
});
💡
Spread Operator (...) Kya Hai?
...req.body ka matlab hai: request body ki saari properties copy kar lo. Jaise { name: "Zaid", email: "z@gmail.com" } — yeh sab new student mein aa jayega!

Lecture 3 — Security

REST API Security — Kyun Zaroori Hai?

Ek unsecured API disaster ho sakta hai. Dekhte hain kya kya threats hain!

💰

Data Chori

Bina security ke hackers user ka poora data chura sakte hain — naam, email, password, credit card sab kuch!

💥

Server Down

Attackers server ko itni requests bhej sakte hain ke woh crash ho jaye — yeh DDoS attack hai!

🔓

Unauthorized Access

Bina authentication ke koi bhi admin panel access kar sakta hai — Bank API imagine karo bina security ke!

🏦
Socho! Banking API Without Security
Agar bank ka API secured na ho toh koi bhi GET /accounts/all karke sabke accounts dekh sakta hai, ya DELETE /accounts/1 karke account delete kar sakta hai!
CORS

CORS — Cross-Origin Resource Sharing

Browser ek important security mechanism enforce karta hai — CORS!

🏠

CORS Kya Hai?

Browser ek security rule follow karta hai: sirf same origin se requests allow karo. Alag domain se request aaye toh server ka permission lena parta hai!

🏫

Real Life Example

Hostel A ke students sirf Hostel A mein ghus sakte hain. Hostel B se koi aaye toh usse permission leni paregi. Browser same kaam karta hai different websites ke liye!

CORS Headers — Server Response 
// Server yeh headers bhejta hai CORS ke liye
Access-Control-Allow-Origin: https://mywebsite.com
Access-Control-Allow-Methods: GET, POST, PUT, DELETE
Access-Control-Allow-Headers: Content-Type, Authorization

// Node.js mein CORS lagana
const cors = require('cors');
app.use(cors({
  origin: 'https://mywebsite.com'  // Sirf yeh domain allow
}));
⚠️
Kabhi Mat Karo — * Use Nahi Karna Sensitive APIs Mein!
Access-Control-Allow-Origin: * matlab hai duniya ka koi bhi domain tumhara API access kar sakta hai. Banking ya health data ke liye yeh bahut dangerous hai!
Attacks

XSS aur DDoS — Do Bade Khatra

Yeh do attacks bahut common hain — samajhna aur bachna zaroori hai!

💉

XSS — Cross-Site Scripting

Attacker website mein malicious script inject karta hai. Jaise comment box mein script likhna — aur woh sab visitors par run ho!

<script>alert("Hacked!")</script>
🌊

DDoS — Distributed Denial of Service

10,000 fake customers dukaan mein ghus aate hain — real customers nahi ghus sakte. Server pe itni requests bhejo ke woh crash ho jaye!

Famous Example: 2016 Dyn attack — Twitter, Netflix, GitHub sab band ho gaye!

🛡

XSS Prevention

Input validate karo. Output encode karo. HTTPOnly cookies use karo. Content Security Policy (CSP) lagao.

🚦

DDoS Protection

Rate limiting lagao (har IP se X requests/minute). CAPTCHA use karo. API Gateway aur CDN use karo.

Auth

Authentication vs Authorization

Yeh dono alag alag hain — bahut log confuse karte hain!

🪪

Authentication (AuthN)

"Tum kaun ho?"

User ki identity verify karna. Username/password, OTP, biometrics — sab authentication ke tareeqe hain.

🎓 University mein ID card check = Authentication
🔑

Authorization (AuthZ)

"Tum kya kar sakte ho?"

User login hai — magar kya access hai usse? Student admin panel nahi dekh sakta, teacher marks upload kar sakta hai.

🔓 Lab mein access milna = Authorization

JWT — JSON Web Token

🎫

JWT Kya Hai?

JWT ek compact token hai authentication ke liye. Teen parts hote hain: Header, Payload, Signature — dots se alag kiye hue.

🔄

JWT Flow

1. User login karta hai. 2. Server JWT generate karta hai. 3. Client token store karta hai. 4. Har request mein token header mein bhejta hai.

🌐

OAuth 2.0

"Login with Google" — yeh OAuth hai! Google, Facebook, Microsoft sab yahi use karte hain. User ka password doosre app ko nahi dena parta.

JWT — Request Header Example 
// Har request mein yeh header bhejo
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

// Node.js mein JWT verify karna
const jwt = require('jsonwebtoken');

const token = req.headers['authorization'].split(' ')[1];
const decoded = jwt.verify(token, 'SECRET_KEY');
// decoded.userId se user identify karo
Sessions

Session Handling in REST

REST stateless hai — toh login kaise yaad rakhein? Yahi challenge hai!

🏛

Traditional Approach (❌ REST ke liye)

Server session store karta hai. Client ke paas sirf Session ID hoti hai cookie mein.

❌ Scalable nahi. Microservices mein problem.
❌ Multiple servers mein kaam nahi karta.
🎫

Token-Based (✅ Recommended)

Server JWT issue karta hai. Client token store karta hai (localStorage/cookie). Har request mein token bhejo.

✅ Stateless — REST ke liye perfect.
✅ Mobile apps ke saath kaam karta hai.
1

User Login Karta Hai

Email + password server ko bhejta hai — POST /api/login

2

Server Verify Karta Hai

Credentials check karta hai database mein

3

JWT Token Milta Hai

Server JWT generate karke client ko bhejta hai

4

Client Token Store Karta Hai

Browser ya mobile mein token save hota hai

5

Har Request Mein Token

Authorization header mein Bearer token bhejta hai

6

Server Verify Karke Access Deta Hai

Token valid hai toh resource milta hai — warna 401 Unauthorized!

📝
Exam Tip — Security Best Practices
HTTPS use karo. HttpOnly cookies. Short token expiry. Refresh tokens. Compromised tokens revoke karo. Yeh 5 points yaad rakhna!
