PART B · 10–35 min

Fetch API — Browser ka Built-in Tool

Fetch browser me pehle se mojood hai — koi library install nahi karni. Yeh ek promise-based function hai jo server se data mangta hai. Simple GET request se lekar POST request tak sab kuch kar sakta hai. Lekin iski kuch limitations hain jo hum seekhenge.

🔍

GET Request Kya Hai?

Server se data manga jata hai. Jaise website khole toh product list ana. Koi data send nahi hota — sirf maanga jata hai.

📤

POST Request Kya Hai?

Server ko naya data bheja jata hai. Jaise registration form submit karna ya naya product add karna.

⚠️

Zarori Baat — response.ok

Fetch sirf network error pe reject karta hai. 404 ya 500 error pe automatically throw NAHI karta. Isliye response.ok check karna zaroori hai.

GET Request — Step by Step (4 Steps)

1

fetch(url) call karo

URL deke request bhejo. Yeh promise return karta hai.

2

response.ok check karo

Agar server ne 200 code bheja toh ok = true. Warna manually error throw karo.

3

response.json() call karo

JSON data ko JavaScript object me convert karo. Yeh bhi async hai, isliye await lagao.

4

State update karo

Data mila — ab useState ya dispatch se UI ko update karo.

JavaScript · Fetch GET Example
// React component me data fetch karna
import { useState, useEffect } from 'react';

function ProductList() {
  const [products, setProducts] = useState([]);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState(null);

  useEffect(() => {
    const fetchProducts = async () => {
      setLoading(true);           // Step 0: loading shuru
      try {
        const res = await fetch('/api/products'); // Step 1: fetch karo

        if (!res.ok) {                // Step 2: check karo
          throw new Error(`HTTP Error: ${res.status}`);
        }

        const data = await res.json(); // Step 3: JSON parse karo
        setProducts(data);            // Step 4: state update karo

      } catch (err) {
        setError(err.message);        // Error handle karo
      } finally {
        setLoading(false);           // Hamesha loading false karo
      }
    };

    fetchProducts();
  }, []);

  if (loading) return <p>Loading...</p>;
  if (error) return <p>Error: {error}</p>;
  return <ul>{products.map(p => <li key={p.id}>{p.name}</li>)}</ul>;
}
✅
Important Note finally block hamesha use karo — chahe success ho ya error, loading ko false karna na bhoolo. Warna spinner hamesha chalta rahega!
⚠️
Common Mistakes 1. await response.json() bhool jana — sirf response.json() likhna kaam nahi karta, yeh Promise return karta hai.   2. POST me Content-Type: application/json header na dena.   3. React StrictMode me effects do baar chalte hain — AbortController se cancel karo.
JavaScript · Fetch POST Example
// POST request — naya user register karna
const registerUser = async (userData) => {
  const res = await fetch('/api/register', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json', // Zaroor dena!
    },
    body: JSON.stringify(userData), // Object ko string banao
  });

  if (!res.ok) throw new Error('Registration failed');
  return res.json();
};
🎯
Exam Tip Theory question me ye zaroor likhna: "Fetch stateless HTTP requests karta hai, JSON responses handle karta hai, aur GET idempotent hota hai jabke POST nahi." HTTP status codes (200, 404, 500) ka zikr karna.
PART C · 35–55 min

Axios — Smarter Library

Axios ek third-party library hai jo npm install axios se install hoti hai. Yeh Fetch se zyada convenient hai — automatically JSON parse karta hai, 404/500 pe error throw karta hai, aur interceptors ki wajah se badi apps me bahut kaam aata hai.

🚀

Auto JSON Parsing

Fetch me res.json() alag se call karna padta tha. Axios me response seedha response.data me aata hai — JSON already parsed!

🔒

Interceptors

Har request ke saath automatically token lagao ya har response me error check karo — ek hi jagah se. Authentication ke liye perfect!

❌

Auto Error on 4xx/5xx

Axios 404 aur 500 pe automatically error throw karta hai. Alag se response.ok check karne ki zaroorat nahi.

JavaScript · Axios Setup + GET + POST
import axios from 'axios';

// ✅ Axios instance create karo baseURL ke saath
const api = axios.create({
  baseURL: 'https://api.myapp.com',
  timeout: 5000,
  headers: { 'Content-Type': 'application/json' },
});

// 🔒 Interceptor — har request me token lagao
api.interceptors.request.use((config) => {
  const token = localStorage.getItem('token');
  if (token) {
    config.headers['Authorization'] = `Bearer ${token}`;
  }
  return config;
});

// GET request — products lao
const getProducts = async () => {
  const response = await api.get('/products');
  return response.data; // ← Seedha data, .json() nahi!
};

// POST request — naya product add karo
const addProduct = async (productData) => {
  const response = await api.post('/products', productData);
  return response.data;
};

// Error handling Axios me
try {
  const data = await getProducts();
} catch (err) {
  if (err.response) {
    // Server ne response bheja (4xx/5xx)
    console.log(err.response.status, err.response.data);
  } else if (err.request) {
    // Request gayi but response nahi aaya (network issue)
    console.log('Network error!');
  }
}
✅
Pro Tip Jab app badi hone lage — multiple pages, auth, retries — tab Axios use karo. Choti app ya sirf ek API call ke liye Fetch kaafi hai.
⚠️
Common Mistakes with Axios 1. baseURL na dena — phir har jagah poora URL likhna padta hai.   2. Error me sirf err.message print karna — err.response aur err.request alag cheezein hain.
COMPARISON

Fetch vs Axios — Side by Side

Dono tools API calls ke liye use hote hain, lekin unke beech key differences hain jo choose karne me madad karti hain.

Feature Fetch API Axios
Browser me Built-in? ✅ Haan ❌ npm install karna hoga
Auto JSON Parse ❌ .json() call karna padta hai ✅ response.data me seedha milta hai
4xx/5xx pe Error ❌ Manually check karo (response.ok) ✅ Auto throw karta hai
Interceptors ❌ Nahi hain ✅ Request/Response dono ke liye
baseURL Config ❌ Har jagah full URL likhna padta hai ✅ Ek baar set, baar baar use
Upload Progress Complicated hai ✅ onUploadProgress event hai
Request Cancel AbortController CancelToken ya AbortController
🎯
Exam Tip — "Why Axios over Fetch?" Ye 4 points zaroor likho: (1) Auto JSON parsing, (2) HTTP error pe auto throw, (3) Interceptors for auth, (4) baseURL config support.
PART E · 65–85 min

Redux — Centralized State

Sochो — aapki app me user, cart, products sab alag-alag components me hain. Har component apna data fetch kare? Ya dono components ke beech prop drilling se share hो? Yahi problem Redux solve karta hai — ek central "store" jahan se poori app data le sakti hai.

🏪

Store

Poori app ka centralized state yahan hota hai. Ek hi store hoti hai puri application me. Koi bhi component isse read kar sakta hai.

⚙️

Reducer

Pure function hai: (state, action) → newState. State directly mutate NAHI karta — naya state return karta hai. API calls bilkul nahi!

📨

Action

Plain JavaScript object hai: { type: 'ADD_ITEM', payload: item }. "Kya hua" batata hai reducer ko.

🔗

React Integration

<Provider store={store}> se app wrap karo. useSelector() se data lo. useDispatch() se action bhejo.

JavaScript · Redux Setup (Classic Way)
// 1. Reducer — pure function, state mutate mat karo!
const initialState = { items: [], loading: false, error: null };

function productsReducer(state = initialState, action) {
  switch (action.type) {
    case 'FETCH_START':
      return { ...state, loading: true, error: null };

    case 'FETCH_SUCCESS':
      return { ...state, loading: false, items: action.payload };

    case 'FETCH_FAIL':
      return { ...state, loading: false, error: action.payload };

    default:
      return state; // Har case me state return karo
  }
}

// 2. Store create karo
import { createStore } from 'redux';
const store = createStore(productsReducer);

// 3. Component me use karo
import { useSelector, useDispatch } from 'react-redux';

function Products() {
  const { items, loading, error } = useSelector(state => state);
  const dispatch = useDispatch();

  return (
    <div>
      <button onClick={() => dispatch({ type: 'FETCH_START' })}>
        Load
      </button>
    </div>
  );
}
🚫
Reducer Me Galti Mat Karna State ko seedha mutate karna sabse bari galti hai: state.items.push(item) — YEH GALAT HAI! Hamesha spread operator use karo: return {...state, items: [...state.items, item]}

✅
Important Note Reducer must be "pure" — matlab: same input pe hamesha same output. Koi API call nahi, koi random nahi, koi side effect nahi. Pure logic only!
PART F · 85–105 min

Redux Thunk — Async Middleware

Problem: Redux reducer me API call nahi ho sakti (pure function hai). Toh API call kahan hogi? Redux Thunk is sawal ka jawab hai! Thunk middleware allow karta hai ke action creator ek function return kare (object ki jagah), aur woh function API call kar sake.

🤔

Thunk Kya Hai?

Ek function jo doosre function ko return kare. Redux Thunk allow karta hai ke dispatch ke saath function bheja jaye, na sirf plain object.

🔄

Async Lifecycle

  • FETCH_START dispatch karo (loading true)
  • API call karo
  • FETCH_SUCCESS ya FETCH_FAIL dispatch karo
🧹

Component Saaf Rehta Hai

Component sirf dispatch(thunk) karta hai. API logic thunk function me hota hai. UI aur logic alag!

JavaScript · Redux Thunk Complete Example
import { createStore, applyMiddleware } from 'redux';
import thunk from 'redux-thunk';

// Store me thunk middleware lagao
const store = createStore(productsReducer, applyMiddleware(thunk));

// ✅ Thunk Action Creator — function return karta hai!
const fetchProducts = () => async (dispatch) => {
  // Step 1: Loading shuru
  dispatch({ type: 'FETCH_START' });

  try {
    // Step 2: API call
    const res = await fetch('/api/products');
    if (!res.ok) throw new Error('Failed!');
    const data = await res.json();

    // Step 3: Success
    dispatch({ type: 'FETCH_SUCCESS', payload: data });

  } catch (err) {
    // Step 4: Failure
    dispatch({ type: 'FETCH_FAIL', payload: err.message });
  }
};

// Component — bilkul saaf!
function ProductPage() {
  const dispatch = useDispatch();
  const { items, loading, error } = useSelector(s => s);

  return (
    <div>
      <button onClick={() => dispatch(fetchProducts())}>
        Load Products
      </button>
      {loading && <p>Loading...</p>}
      {error && <p>Error: {error}</p>}
      {items.map(p => <div key={p.id}>{p.name}</div>)}
    </div>
  );
}

RECAP

5 Key Takeaways

01 Fetch built-in hai lekin JSON aur errors manually handle karne padte hain.
02 Axios cleaner API deta hai — interceptors, auto JSON, baseURL config.
03 Redux shared state ko centralize karta hai — predictable updates ke liye.
04 Thunk async logic ko Redux me possible banata hai — middleware ke zariye.
05 Har async operation ko 3 states se model karo: loading, data, error.

3 Viva Questions — Practice karo

❓

Q1: Fetch 404 pe error kyun nahi throw karta?

Jawab: Fetch sirf network-level failures pe reject karta hai (jaise server mil hi nahi). HTTP status codes (404, 500) ko woh "successful" response maanta hai — response aaya toh promise resolve hoga. Isliye response.ok check karna zaroori hai.

❓

Q2: Redux me Middleware ka kya kaam hai?

Jawab: Middleware dispatch aur reducer ke beech mein hota hai. Yeh actions ko intercept karta hai. Thunk middleware check karta hai ke agar action ek function hai, toh use run karo; agar plain object hai, toh seedha reducer ko bhejo.

❓

Q3: Reducer me side effects kyun nahi honi chahiye?

Jawab: Redux ka core principle "predictability" hai. Agar reducer me API calls ya random values houn toh same input pe alag output aa sakta hai, jo debugging mushkil aur time-travel debugging impossible kar deta hai. Pure functions hamesha same output dete hain.
