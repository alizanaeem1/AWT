Decorators — Labelling System
Angular mein class ko "kaun hai yeh" batane ka tarika

Socho ek school mein students hain — kisi ke ID card par likha hai "Teacher", kisi par "Student", kisi par "Principal". Angular mein bhi yahi kaam Decorator karta hai. Yeh TypeScript ka ek special feature hai jo class ya function ke upar @ sign se likha jaata hai aur Angular ko batata hai — "Yeh class kya hai aur kya karti hai."

🧩

@Component

Batata hai ke yeh class ek UI Component hai. Iske saath template (HTML) aur style (CSS) attach hoti hai.

📦

@NgModule

Puri application ko organize karta hai. Modules ke andar Components, Services, aur Imports define hote hain.

🔧

@Injectable

Service class par lagata hain — batata hai ke yeh class dusri jagah inject ho sakti hai (use ho sakti hai).

📥

@Input / @Output

Parent se child component mein data bhejna (@Input) ya child se parent ko event bhejne ke liye (@Output).

📄 app.component.tsTypeScript
// @Component decorator — class ko component banata hai
import { Component } from '@angular/core';

@Component({
  selector: 'app-root',       // HTML tag naam
  templateUrl: './app.component.html',  // HTML file
  styleUrls: ['./app.component.css']   // CSS file
})
export class AppComponent {
  title = 'Meri Pehli Angular App';
}
💡 Yaad Rakho (Tip)

Decorator hamesha @ sign se shuru hota hai aur class se upar likha jaata hai. Yeh class ke behaviour ko define karta hai — class khud nahi karta.

📝 Exam Tip

Exam mein poochhein: "Angular mein Decorator kya hai?" — Jawab: Yeh TypeScript metadata hai jo class ko Angular ke liye label karta hai. @Component, @NgModule, @Injectable common examples hain.

Modules — Application ki Almari
Angular app ko organize karne ka tarika

Socho tumhare ghar mein ek bari almari hai. Us almari mein alag alag drawers hain — ek kapdon ke liye, ek kitabon ke liye, ek khilonon ke liye. Angular mein Module wahi almari ka kaam karta hai. Har module mein related components, services, aur pipes hoti hain.

📋 NgModule ke 4 main properties:

  • declarations: Woh components/directives/pipes jo is module ke hain
  • imports: Dusre modules jo is module ko chahiye (jaise BrowserModule, FormsModule)
  • providers: Services jo poori app mein use hongi
  • bootstrap: Woh root component jo pehle load hoga (sirf AppModule mein)
📄 app.module.tsTypeScript
import { NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { FormsModule } from '@angular/forms';
import { AppComponent } from './app.component';

@NgModule({
  declarations: [AppComponent],  // Hamare components
  imports: [
    BrowserModule,              // Browser support ke liye
    FormsModule                 // Form handling ke liye
  ],
  providers: [],                // Services yahan aati hain
  bootstrap: [AppComponent]  // Pehle yeh load hoga
})
export class AppModule { }

Components — UI ke Building Blocks
Angular app LEGO jaisi hoti hai — components se banti hai

Imagine karo ek website — uska header alag component hai, footer alag, sidebar alag, aur main content alag. Jaise LEGO blocks mila kar ek bari cheez banti hai, waise Angular components mila kar puri app banti hai. Har component mein 3 cheezein hoti hain:

📄

Template (HTML)

Component ki shakal — woh jo user screen par dekhta hai. Angular ke special syntax (.html file) se banana hota hai.

⚙️

Class (TypeScript)

Component ka dimag — data aur logic yahan hota hai. Variables, functions sab yahan define hote hain.

🎨

Styles (CSS)

Component ki sajawat — sirf is component par apply hoti hai, dusri jagah nahi (scoped CSS).

Component Life Cycle — Zindagi ke Stages

Har component janm leta hai, barhta hai, aur khatam hota hai. Angular ke lifecycle hooks yeh stages manage karte hain:

  • 1

    ngOnInit()

    Component banne ke baad PEHLI baar chalta hai. API calls yahan karo. Constructor ke baad aata hai.

  • 2

    ngOnChanges()

    Jab bhi @Input property change ho, yeh chalta hai. Parent se nayi value aayi? Yahan pata chalega.

  • 3

    ngDoCheck()

    Har change detection cycle mein chalta hai. Custom change detection ke liye use hota hai.

  • 4

    ngOnDestroy()

    Component hatne se PEHLE chalta hai. Subscriptions unsubscribe karo, memory clean karo yahan.

📄 student.component.tsTypeScript
import { Component, OnInit, Input } from '@angular/core';

@Component({
  selector: 'app-student',
  template: `
    <div class="student-card">
      <h2>{{ studentName }}</h2>
      <p>Roll No: {{ rollNo }}</p>
    </div>
  `
})
export class StudentComponent implements OnInit {
  @Input() studentName: string = '';
  rollNo: number = 101;

  ngOnInit(): void {
    console.log('Component tayyar ho gaya!');
    // Yahan API call karo
  }
}

Data Binding — Data ka Safar
TypeScript class aur HTML template ke darmiyan data share karna

Data Binding matlab hai — TypeScript (class) mein jo data hai woh HTML (template) mein dikhe, aur user jo HTML mein kare woh class mein pohanche. Yeh ek do-tarfah raasta hai.

TypeSyntaxDirectionExample
Interpolation{{ value }}Class → Template{{ title }}
Property Binding[property]="value"Class → Template[src]="imageUrl"
Event Binding(event)="method()"Template → Class(click)="save()"
Two-Way Binding[(ngModel)]="var"Dono taraf[(ngModel)]="name"
📄 data-binding.component.htmlHTML + Angular
<!-- 1. Interpolation — class ka data HTML mein dikhao -->
<h1>{{ pageTitle }}</h1>
<p>Student ka naam: {{ studentName }}</p>

<!-- 2. Property Binding — HTML property set karo -->
<img [src]="profilePic" [alt]="studentName">
<button [disabled]="isLoading">Save</button>

<!-- 3. Event Binding — user action pe function call karo -->
<button (click)="onSubmit()">Submit</button>
<input (keyup)="onSearch($event)">

<!-- 4. Two-Way Binding — input aur display dono saath -->
<input [(ngModel)]="searchText" placeholder="Search karo...">
<p>Tum likhe: {{ searchText }}</p>
⚠️ Dhyan Do

Two-way binding [(ngModel)] use karne ke liye FormsModule ko AppModule ke imports array mein zaror add karo, warna error aayega!

Templating — Dynamic HTML Banana
Angular ka powerful HTML syntax system

Normal HTML static hoti hai — ek baar likh do, wahi dikhti hai. Angular ka templating system HTML ko zinda bana deta hai! Data change ho, HTML bhi change ho jaye. Conditions lagao, loops chalao — sab template mein ho sakta hai.

🔀

*ngIf — Agar/Warna

Kisi cheez ko condition ke basis par dikhao ya chhupao. JavaScript ke if/else jaisa kaam karta hai HTML mein.

🔁

*ngFor — List Banana

Array ke har item ke liye HTML element banana. Shopping list, student list — sab isi se banta hai.

🎛

ngSwitch — Multiple Cases

Ek value ke hisaab se alag alag content dikhao. Switch statement jaisa kaam karta hai HTML mein.

📄 template-examples.htmlAngular Template
<!-- ngIf — condition ke basis par dikhao -->
<div *ngIf="isLoggedIn">
  <p>Welcome! Aap logged in hain.</p>
</div>
<div *ngIf="!isLoggedIn">
  <p>Pehle login karein.</p>
</div>

<!-- ngFor — list ke har item ke liye -->
<ul>
  <li *ngFor="let student of students; let i = index">
    {{ i + 1 }}. {{ student.name }} — Roll: {{ student.roll }}
  </li>
</ul>

<!-- ngSwitch — multiple conditions -->
<div [ngSwitch]="userRole">
  <p *ngSwitchCase="'admin'">Admin Panel</p>
  <p *ngSwitchCase="'student'">Student Dashboard</p>
  <p *ngSwitchDefault>Guest View</p>
</div>

Directives — HTML ko Superpowers Dena
HTML elements ke behaviour aur appearance ko badalna

Directives HTML elements ko extra powers deti hain. Jaise ek normal insaan ko superhero suit pehna do — woh insaan to wahi hai lekin ab us mein naye powers aa gaye hain! Angular mein 3 qism ki directives hain:

🏗️

Structural Directives

DOM structure badal deti hain — elements add ya remove karti hain. *ngIf, *ngFor, *ngSwitch examples hain. * se shuru hoti hain.

🎨

Attribute Directives

Element ki appearance ya behaviour badal deti hain. ngClass, ngStyle, ngModel examples hain.

🔧

Custom Directives

Khud ki apni directive bana sakte ho. Jaise "highlighted" directive — jo kisi bhi element ko hover par highlight kare.

📄 directives-example.html + tsAngular
<!-- ngClass — condition ke basis par CSS class lagao -->
<div [ngClass]="{'active': isActive, 'error': hasError}">
  Yeh div hai
</div>

<!-- ngStyle — inline style dynamically lagao -->
<p [ngStyle]="{'color': textColor, 'font-size': fontSize + 'px'}">
  Colored text
</p>

// Custom Directive banana:
import { Directive, ElementRef, HostListener } from '@angular/core';

@Directive({ selector: '[appHighlight]' })
export class HighlightDirective {
  constructor(private el: ElementRef) {}

  @HostListener('mouseenter') onHover() {
    this.el.nativeElement.style.background = 'yellow';
  }
  @HostListener('mouseleave') onLeave() {
    this.el.nativeElement.style.background = '';
  }
}

Pipes — Data ko Format Karna
Data display karne se pehle usko khubsoorat banana

Socho tumhare paas ek rough diamond hai — seedha dukaan mein rakh do to koi nahi kharidega. Pehle use polish karo, cut karo — phir khubsoorat dikhega. Pipe bhi yahi karta hai data ke saath. Data class mein ho sakta hai raw format mein, lekin HTML mein display karte waqt Pipe use karke use format kar dete hain.

PipeKya karta haiExampleOutput
uppercaseSab bade letters{{ 'hello' | uppercase }}HELLO
lowercaseSab chhote letters{{ 'HELLO' | lowercase }}hello
dateDate format karo{{ today | date:'dd/MM/yyyy' }}08/05/2026
currencyCurrency format{{ 1500 | currency:'PKR' }}PKR 1,500
jsonObject ko JSON string{{ obj | json }}{ "a": 1 }
sliceArray/string cut karo{{ arr | slice:0:3 }}Pehle 3 items
📄 pipes-example.htmlAngular Template
<p>{{ studentName | uppercase }}</p>
<p>{{ fees | currency:'PKR':'symbol':'1.0-0' }}</p>
<p>{{ admissionDate | date:'fullDate' }}</p>
<p>{{ description | slice:0:100 }}...</p>

<!-- Pipe chain karna — ek ke baad ek -->
<p>{{ studentName | lowercase | titlecase }}</p>
💡 Custom Pipe bhi bana sakte hain

Jaise @Pipe({ name: 'rupees' }) decorator se apna custom pipe banao aur {{ 5000 | rupees }} use karo. PipeTransform interface implement karna hota hai.

Dependency Injection — Smart Way to Share
Objects aur services ko manage karne ka Angular ka system

Imagine karo ek hospital mein — doctor ko oxygen cylinder chahiye. Doctor khud factory se cylinder nahi laata — hospital us ko provide karta hai. Angular mein bhi yahi hota hai. Component ko agar Service chahiye, woh khud nahi banata — Angular ka Injector automatically provide karta hai. Isi ko Dependency Injection kehte hain.

  • 1

    Service banao

    @Injectable({ providedIn: 'root' }) lagao — yeh service puri app mein available ho jaayegi.

  • 2

    Constructor mein inject karo

    Component ke constructor mein service type likhdo — Angular automatically object de dega.

  • 3

    Use karo

    Ab this.myService.method() se service ke functions use kar sakte ho.

📄 DI ExampleTypeScript
// Step 1: Service banao
@Injectable({ providedIn: 'root' })
export class StudentService {
  getStudents() {
    return [{ name: 'Ali' }, { name: 'Sara' }];
  }
}

// Step 2: Component mein inject karo
@Component({ selector: 'app-list', template: '...' })
export class ListComponent implements OnInit {
  students = [];

  // Constructor mein inject karo — Angular dega automatically
  constructor(private studentService: StudentService) {}

  ngOnInit() {
    // Step 3: Use karo
    this.students = this.studentService.getStudents();
  }
}
⚙️
Services — Shared Logic ka Ghar
Business logic aur data sharing ke liye reusable classes

Socho ek school mein library hai. Har student apni kitabein ghar nahi rakhta — sab library se lete hain. Angular mein Service wahi library hai. Koi bhi component service se data le sakta hai, dono ke darmiyan share ho sakta hai. HTTP calls, business logic — sab services mein hoti hai.

📄 data.service.tsTypeScript
import { Injectable } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { Observable } from 'rxjs';

@Injectable({ providedIn: 'root' })
export class DataService {
  private apiUrl = 'https://api.example.com/students';

  constructor(private http: HttpClient) {}

  // Server se students laao
  getStudents(): Observable<any[]> {
    return this.http.get<any[]>(this.apiUrl);
  }

  // Naya student add karo
  addStudent(student: any): Observable<any> {
    return this.http.post(this.apiUrl, student);
  }
}
📝 Exam Tip

Service aur Component ka farq: Component sirf UI handle karta hai. Service business logic, API calls, data sharing karta hai. Service ko multiple components share kar sakte hain — yeh code reuse ka best practice hai.

Routing — Page Navigation System
Multiple pages ke darmiyan navigate karna bina page reload ke

Normal websites mein page click karo to pura page reload hota hai. Angular mein Single Page Application (SPA) hoti hai — page reload nahi hota, sirf content change hota hai. Router yeh kaam karta hai. URL change hoti hai lekin page nahi.

📄 app-routing.module.tsTypeScript
import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';
import { HomeComponent } from './home/home.component';
import { StudentsComponent } from './students/students.component';
import { AboutComponent } from './about/about.component';

const routes: Routes = [
  { path: '', component: HomeComponent },           // /
  { path: 'students', component: StudentsComponent }, // /students
  { path: 'student/:id', component: DetailComponent},// /student/5
  { path: 'about', component: AboutComponent },       // /about
  { path: '**', component: NotFoundComponent }        // 404 page
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppRoutingModule { }

Observable & RxJS — Async Data ka Safar
Future mein aane wale data ko handle karna

Socho tumne pizza order kiya. Pizza abhi nahi aaya lekin tum intezaar kar rahe ho. Jab pizza aayega, tum khaoge. Observable bhi aisa hi hai — yeh ek "future mein aane wala data" hai. Jab API se data aayega, Observable usse deliver karega. Tum usse subscribe karke wait karte ho.

📡

Observable

Data ka stream — ek ya zyada values future mein deliver ho sakti hain. HTTP calls Observable return karte hain.

👂

Subscribe

Observable ko "sun'na" shuru karna. Jab data aaye, kya karna hai — woh yahan define karo.

🔀

Operators (RxJS)

Data ko filter, transform, combine karne ke tools. map, filter, switchMap common hain.

📄 observable-example.tsTypeScript + RxJS
import { Component, OnInit, OnDestroy } from '@angular/core';
import { DataService } from './data.service';
import { Subscription } from 'rxjs';
import { map, filter } from 'rxjs/operators';

export class MyComponent implements OnInit, OnDestroy {
  students = [];
  private sub: Subscription;

  constructor(private dataService: DataService) {}

  ngOnInit() {
    // Subscribe karo — pizza ka intezaar karo!
    this.sub = this.dataService.getStudents()
      .pipe(
        map(data => data.filter(s => s.active)), // sirf active students
        map(data => data.slice(0, 10))           // sirf pehle 10
      )
      .subscribe({
        next: (data) => { this.students = data; },  // data aaya
        error: (err) => { console.error(err); },    // error aai
        complete: () => { console.log('Done!'); }   // khatam hua
      });
  }

  ngOnDestroy() {
    this.sub.unsubscribe(); // Memory leak rokne ke liye
  }
}
⚠️ Memory Leak — Bari Ghalti

Observable subscribe karo to ngOnDestroy() mein unsubscribe() zaroor karo. Warna component khatam hone ke baad bhi Observable chalta rahega — yeh memory leak hai aur app slow ho jaayegi!
