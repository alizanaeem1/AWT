OOP Kya Hota Hai?
Object-Oriented Programming — Basics

OOP (Object-Oriented Programming) programming ka ek tarika hai jisme hum duniya ki cheezein ko objects ki tarah sochte hain. Jaise real life mein ek "car" hoti hai jiske properties hote hain (color, brand) aur kaam bhi (chalana, rokna) — bilkul waise TypeScript mein bhi hum objects banate hain.

🏗️
Class
Blueprint ya template. Jaise architect ka naksha jo ke building banane ke liye use hota hai. Ek class se hazaron objects ban sakte hain.
📦
Object
Class ka real instance. Jaise naksha se bani asli building. Har object ki apni values hoti hain.
🔐
Encapsulation
Data ko andar chhupaana. Jaise ATM machine — andar ka circuit nahi dikhta, sirf buttons aur screen dikhte hain.
🧬
Inheritance
Ek class doosri class ke properties aur methods le sakti hai. Jaise bacha parents se traits inherit karta hai.
💡
Tip TypeScript JavaScript ke upar bana hai lekin usme Types aur OOP features zyada powerful hain. Java jaisi class support milti hai TypeScript mein!

Classes in TypeScript
Part 1 · 0–30 min

Class ek blueprint hai jisse hum objects banate hain. Socho jaise ek Student ka form template hota hai — usme fields hote hain (naam, age, roll number). Jab koi student form bharta hai, ek naya object banta hai usi template se.

Class mein 3 zaroori cheezein hoti hain:

📝
Properties
Class ke variables. Jaise name, age, balance. Object ka data yahan store hota hai.
🔧
Constructor
Khaas function jo object banate waqt ek baar chalta hai. Properties ko starting values deta hai.
⚙️
Methods
Class ke andar ke functions. Object kya kaam karta hai woh yahan define hota hai.
TypeScript — Student Class
// Class banana — Student ka blueprint class Student { // Properties (class ke variables) name: string; age: number; // Constructor — object banate waqt chalta hai constructor(name: string, age: number) { this.name = name; this.age = age; } // Method — student ki info dikhao display(): void { console.log(`Naam: ${this.name}, Umar: ${this.age}`); } } // Object banana (class use karna) const s1 = new Student("Ali", 20); const s2 = new Student("Sara", 22); s1.display(); // Output: Naam: Ali, Umar: 20 s2.display(); // Output: Naam: Sara, Umar: 22
ℹ️
Yaad Rakho new keyword se object banta hai. this keyword current object ko refer karta hai — matlab "is class ke is object ki property."

Real-life example — Bank Account:

TypeScript — BankAccount Class
class BankAccount { accountNumber: string; balance: number; constructor(accNum: string, initialBalance: number) { this.accountNumber = accNum; this.balance = initialBalance; } deposit(amount: number): void { this.balance += amount; console.log(`Jamah: ${amount}. Baqi: ${this.balance}`); } withdraw(amount: number): void { if (amount > this.balance) { console.log("Paisay kam hain!"); } else { this.balance -= amount; } } } const acc = new BankAccount("PKR-001", 5000); acc.deposit(2000); // Jamah: 2000. Baqi: 7000 acc.withdraw(1000); // Balance: 6000

Access Modifiers
public · private · protected

Access Modifiers control karte hain ke class ki properties aur methods bahar se access ho sakti hain ya nahi. Jaise ghar mein kuch cheezein sab ke liye hoti hain (drawing room), kuch sirf family ke liye (bedroom), aur kuch sirf aur bachon ke saath share hoti hain (family secrets).

ModifierKahan Access?Real-life Misal
public Har jagah — class ke andar, bahar, child mein Ghar ka drawing room — sab aa sakte hain
private Sirf usi class ke andar Tizori (safe) — sirf maalik access kar sakta hai
protected Class + uski child classes mein Family secret — sirf family jaanti hai
Access Modifiers Example
class Person { public name: string; // Sab access kar sakte hain private password: string; // Sirf is class mein protected salary: number; // Class + children constructor(name: string, pass: string, sal: number) { this.name = name; this.password = pass; this.salary = sal; } } const p = new Person("Ali", "abc123", 50000); console.log(p.name); // ✅ Theek — public hai // console.log(p.password); ❌ Error — private hai! // console.log(p.salary); ❌ Error — protected hai!
⚠️
Common Mistake Private property ko bahar access karne ki koshish karna TypeScript mein compile-time error deta hai. Agar access chahiye to getter/setter method banao.
🎯
Exam Tip "private vs protected ka farq" aksar exam mein aata hai. Yaad rakho: private = sirf woh class, protected = woh + uski children.

Inheritance — Wiraasat
Part 2 · 30–60 min

Inheritance ka matlab hai ek class doosri class se properties aur methods le sakti hai — dobara likhe bina! Jaise bacha apne walidain se kuch khasiyatein inherit karta hai (aankhen, rang, qad), bilkul waise child class apni parent class se code inherit karti hai.

Vehicle (Parent)
move(), fuel
→
Car (Child)
doors, honk()
→
ElectricCar (Grandchild)
battery, charge()
Inheritance — extends keyword
// Parent class class Animal { name: string; constructor(name: string) { this.name = name; } move(): void { console.log(`${this.name} chal raha hai...`); } } // Child class — Animal se inherit karta hai class Dog extends Animal { // ⭐ super() parent ka constructor call karta hai constructor(name: string) { super(name); } // Dog ka apna method bark(): void { console.log(`${this.name} bhaunk raha hai!`); } } const dog = new Dog("Tommy"); dog.move(); // ✅ Parent se mila — Tommy chal raha hai... dog.bark(); // ✅ Apna — Tommy bhaunk raha hai!

Method Overriding — Parent ka method badalna:

Kabhi kabhi child class chahti hai ke parent ka method usi naam se ho lekin kaam alag kare. Isay Method Overriding kehte hain.

Method Overriding
class Animal { speak(): void { console.log("Animal koi awaaz karta hai"); } } class Cat extends Animal { // ⭐ Override — same naam, alag kaam speak(): void { console.log("Billi: Meow Meow! 🐱"); } } class Dog extends Animal { speak(): void { console.log("Kutta: Woof Woof! 🐶"); } } const cat = new Cat(); const dog = new Dog(); cat.speak(); // Billi: Meow Meow! dog.speak(); // Kutta: Woof Woof!
♻️
Code Reuse
Ek baar likha code baar baar use hota hai. Parent class ka method child apne aap use kar sakti hai.
🧹
Saaf Structure
Code zyada organized hota hai. Related cheezein ek jagah hoti hain — easier to understand.
🔧
Easy Maintenance
Parent mein ek jagah change karo — sab children mein automatically apply ho jaata hai.
🎯
Exam Tip extends keyword inheritance ke liye use hota hai. super() parent constructor call karta hai — child constructor mein super() pehle aana chahiye!

Interfaces — Contract
Part 3 · 60–90 min

Interface ek contract (samjhauta) hai jo define karta hai ke class mein kya hona chahiye — lekin kaise nahi batata. Socho jaise ek naukri ka contract: "Aapko yeh kaam karna hai" — procedure aap decide karo.

📄
Naukri Contract
Contract kehta hai: "Clerk ko attendance register karna hai, filing karni hai." — Kaise karna hai woh clerk decide karta hai.
🔌
USB Standard
USB port ka standard define hai — size, pins. Koi bhi company USB device bana sakti hai us standard ko follow karke.
📐
Shape Interface
Agar "Shape" interface area() maange, to Circle, Rectangle, Triangle sab apne hisaab se area() calculate karein — formula alag hoga.
Interface Syntax + implements
// Interface — sirf structure define karta hai interface Shape { color: string; area(): number; // Koi implementation nahi! describe(): void; } // Circle class Shape interface follow karti hai class Circle implements Shape { color: string; radius: number; constructor(color: string, radius: number) { this.color = color; this.radius = radius; } // Interface ne kaha tha area() banao — yahan banaya area(): number { return Math.PI * this.radius ** 2; } describe(): void { console.log(`${this.color} circle, area: ${this.area().toFixed(2)}`); } } const c = new Circle("Laal", 5); c.describe(); // Laal circle, area: 78.54

Multiple Interfaces — Ek class kai interfaces follow kar sakti hai:

Multiple Interfaces
interface Printable { print(): void; } interface Saveable { save(): void; } // ⭐ Dono interfaces ek saath implement ki! class Document implements Printable, Saveable { title: string; constructor(title: string) { this.title = title; } print(): void { console.log(`Printing: ${this.title}`); } save(): void { console.log(`Saving: ${this.title}`); } } const doc = new Document("Meri Report"); doc.print(); // Printing: Meri Report doc.save(); // Saving: Meri Report
⚠️
Important Interface mein koi implementation nahi hoti — sirf structure. Agar interface mein koi method likha aur class mein implement nahi kiya to TypeScript error dega!

Class vs Interface
Part 4 · 90–120 min

Ab dono mein farq samajhte hain. Yeh table bahut zaroori hai — exam mein zaroor aata hai!

FeatureClassInterface
Kaam (Purpose) Blueprint + Implementation Sirf Contract/Structure
Object Banana Haan — new keyword se Nahi
Constructor Hota hai Nahi hota
Implementation Methods ka code hota hai Sirf signature (naam)
Inheritance extends — sirf ek extends — kai sare
Multiple Follow Ek se extend — Multiple nahi Kai interfaces implement ho sakte hain
Keyword extends implements
🏗️
Class kab use karein?
Jab aapko actual implementation chahiye — methods ka kaam karna bhi likhna ho. Jab object banana ho. Jab constructor chahiye.

Example: Student, BankAccount, Car
📋
Interface kab use karein?
Jab sirf structure ya contract define karna ho. Jab multiple types ek jesi structure follow karain. Jab flexibility chahiye.

Example: Shape, Printable, Serializable

Combined Example — tینوں concepts ek saath:

Interface + Class + Inheritance
// Interface — Vehicle ke rules interface VehicleRules { fuel: string; startEngine(): void; } // Base class — interface implement karta hai class Car implements VehicleRules { fuel: string; brand: string; constructor(brand: string, fuel: string) { this.brand = brand; this.fuel = fuel; } startEngine(): void { console.log(`${this.brand} engine start — ${this.fuel} se`); } } // Child class — Car se inherit karta hai class ElectricCar extends Car { battery: number; constructor(brand: string, batteryKwh: number) { super(brand, "Electric"); // Parent constructor call this.battery = batteryKwh; } charge(): void { console.log(`Charging ${this.battery} kWh battery...`); } } const tesla = new ElectricCar("Tesla", 100); tesla.startEngine(); // Tesla engine start — Electric se tesla.charge(); // Charging 100 kWh battery...
